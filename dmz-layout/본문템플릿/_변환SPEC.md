## 변환 규칙 (SPEC)

1. **산출물 범위**: 흰 박스(`detail-body-wrap`) **inner HTML만**. 흰 박스 wrapper·핑크 헤더(태그·메타데이터)는 동현공 chrome이므로 **포함하지 않는다.** 템플릿 파일의 `<!-- renderedBodyHtml START -->` ~ `END` 사이가 정확히 산출 범위.
2. **스타일**: 전부 인라인(`dangerouslySetInnerHTML` 자족). 클래스는 폰트 클래스(`handwriting`/`typewriter`)와 빈칸(`dmz-blank`)만 사용.
3. **빈칸**: MD 본문의 `[A]` `[B]` `[C]` 마커를 그대로 cyan span으로:
   `<span class="dmz-blank" data-blank="A" style="display:inline-block; padding:0 0.5em; background:#3FE0DC; border-radius:4px; color:#1a2b4a;">[A]</span>`
   정답 텍스트는 넣지 않는다(게임 코드가 채움). 마커만 보존.
4. **색 토큰**: 핑크 #FF6EC7 / cyan #3FE0DC / 네이비 #1a2b4a / 그레이 #6E6E6E
5. **폰트 클래스**: 손글씨=`handwriting`, 타자기류=`typewriter`, 기본=Paperlogy(클래스 없음). 템플릿 따름.
6. **본문 충실**: MD 본문 텍스트를 누락 없이 옮긴다. 제목·문단·서명·날짜 등 구조 유지. MD의 `**볼드**`는 템플릿 결에 맞게 볼드 처리.
7. **사진 타입**: 사진 URL = `https://res.nolgong.com/dmz-archive/` + CSV 사진 컬럼 파일명. 단 백도는 사진 파일명을 모르므로, 사진 타입은 `03-사진.html` 구조를 따르되 `<img src>` 자리에 `{{PHOTO_URL}}` placeholder를 넣는다 (메인이 CSV 사진 컬럼으로 치환). 2장짜리는 placeholder 2개.

## 출력

`Assets/incoming/통일부/본문 데이터 HTML/clean/out_{주제영문}.html` 파일 하나에 모든 자료를 아래 구분자로 이어 붙인다:

```
<!--ITEM|{스토리}|{타이틀}|{타입명}-->
{renderedBodyHtml — 흰 박스 inner HTML}
<!--/ITEM-->
```

- 스토리·타이틀은 MD 헤더 그대로 (CSV 매칭 키). 파이프 `|`는 구분자이므로 값 안에 쓰지 말 것.
- 자료 순서는 MD 등장 순서.
- 파일 상단에 `<!-- {주제} 자료 N건 -->` 한 줄.

## 검증 (백도 자체)

- 모든 자료가 ITEM 블록으로 출력됐는지 (개수 = MD 헤더 개수)
- 빈칸 마커가 cyan span으로 빠짐없이 변환됐는지
- 한글 NFC·제어문자(`\x08` 등) 없는지
- 완료 후: 처리한 자료 개수 + 타입별 분포 + 출력 파일 경로 한 줄 보고

## 영문/버전 재사용 가이드

이 파이프라인은 **입력 텍스트만 바꾸면 재사용**된다. 다음 작업(같은 버전 영문)을 위해 변경 지점을 고정해 둔다.

**버전 교체 시 (한글, 예: 260529 → 다음)**
1. 원본 md를 폴더에 추가 → `clean/{버전}_통일부_{주제}.md`로 base64 제거(`grep -av "data:image\|base64"`)
2. 매칭 키 안정성 점검: 새 헤더(스토리/타이틀/타입)가 마스터 `사진링크용.csv`와 일치하는지 diff. **타이틀이 바뀌면 merge 매칭이 깨진다** — 바뀌었으면 CSV 타이틀도 함께 갱신
3. 이 SPEC §입력 경로 + 버전 이력 갱신 → 백도 변환 → merge/verify(산출 CSV는 `_{버전}` suffix로 보존)

**빈칸 마커**: `[A][B][C]` 규칙·cyan span은 언어 무관 동일

### 영문판 실행 명세 (6/5 세션432 확정 — 한글 260529 골격 재사용)

한글판과 결정적으로 다른 점: 이번엔 정리된 clean MD를 받지 않고 **docx 원본에서 영문 본문을 골라내야** 한다. 그래서 "영작"이 아니라 "걸러내기" 작업이다. docx 안에 `영문(클린본)`이 자료별로 이미 정리돼 있어 그것을 그대로 옮긴다.

**입력**
- 영문 텍스트 소스: `영문번역/{NN. 주제}/*.docx` (영한 대역, 4차검수완, 축약본). 스토리 1개 = docx 1개 = 자료 4개(A/B/C/D)
- docx → txt 변환본: `영문번역/txt/{NN. 주제}.txt` (메인이 textutil로 사전 변환, 코드순)
- **구조 골격 = 한글 260529 `clean/out_{주제}.html`** — ITEM 블록의 HTML 구조(div/p/span/img placeholder)를 그대로 미러. 백도는 이 HTML을 받아 **본문 텍스트만 영문으로 교체**한다(템플릿 재해석 불필요, 한글과 100% 동일 모양 보장)

**제외 규칙 (피터공 3규칙 — docx에만 있는 군더더기. 한글 클린본이 이미 같은 모양으로 깎여 있음)**
1. `빈칸 정답표 / Answer Key` 표 전체 제외
2. `자료 A`·`Source A (Letter)`·`지식인 QnA` 등 기록 구분 요소/라벨 제외
3. 출처·날짜·구술자·보내는이/받는이 등 메타데이터 제외 (= 한글 본문이 이미 뺀 것들)
- 유지: 제목(=타이틀, 흰박스 헤딩) + 본문 + `[A][B][C][D]` 빈칸 cyan span + 사진 `{{PHOTO_URL}}` placeholder
- 축약본 우선: docx에 전문/축약 두 버전이 섞이면 축약(클린본) 채택. 영문 `영문(클린본)`이 곧 축약 반영본

**출력**
- `clean/out_{주제}_en.html` — ITEM 구분자 형식 동일. **ITEM 헤더(스토리\|타이틀\|타입)는 한글 그대로**(매칭키 보존), 본문만 영문
- merge: `DMZ_VERSION=EN DMZ_OUT_SUFFIX=_en python3 merge.py` → `사진링크용_본문채움_EN.csv` (마스터 한글 CSV에 영문 본문 idx6 채움+사진 치환)
- verify/preview: `DMZ_VERSION=EN python3 verify.py` / `... preview_csv.py`(→ `clean/preview_EN.html`)

**전달**: 한글판과 동일 흐름. 동현공께는 별도 EN CSV의 본문 컬럼(idx6)만 영문 시트로 복제. 매칭키·사진·타이틀은 한글 CSV 그대로라 한영 행이 1:1 대응.

**템플릿**: design_sample에 영문 전용 샘플 없음 → 14종 템플릿 그대로 재사용(폰트는 동현공 임베드 영역). 한글 out html을 골격으로 쓰므로 백도는 템플릿을 다시 읽지 않아도 된다.

## 백도 위임 프롬프트 (템플릿)

주제 5개를 각각 백도 1개(sonnet)에 위임한다. 본문 변환 규칙은 백도가 이 SPEC을 Read하므로 프롬프트에 중복하지 않고, 아래 4개 변수만 바꾼다: `{주제}` · `{버전}` · `{out파일}` · `{N}`(자료 개수). 주제↔out파일·자료개수는 `docs/SPEC.md` §데이터 계약 참조.

```
이 인스턴스는 아리공(볼트 담당, /Users/p.air15/Neo-Obsi-Sync). 볼트 .md 수정 가능.
DMZ 자료 본문을 디자인 템플릿대로 renderedBodyHtml로 변환하는 작업(버전 {버전}).
[필수1] 변환 SPEC Read: /Users/p.air15/Neo-Obsi-Sync/_dev/dmz-layout/본문템플릿/_변환SPEC.md
[필수2] 입력에 등장하는 각 타입의 템플릿(_dev/dmz-layout/본문템플릿/NN-*.html) Read해서 그 구조·인라인 스타일 그대로 따름.
입력: Assets/incoming/통일부/본문 데이터 HTML/clean/{버전}_통일부_{주제}.md
출력: 같은 clean/ 폴더에 {out파일} (SPEC §출력 ITEM 구분자 형식 그대로)
이 입력엔 자료 {N}개 (반드시 {N}개 ITEM 출력).
변환 규칙 엄수: 흰 박스 inner HTML만 / 인라인 스타일 / 빈칸 [A][B][C]→cyan span(정답 텍스트 없음) /
색 토큰 / 폰트 클래스 / 사진 <img src> 자리에 {{PHOTO_URL}} placeholder(2장이면 2개) /
본문 텍스트 누락 없이 충실(★원문 어휘를 동의어로 바꾸거나 요소 누락 금지).
완료 보고: out 경로 + 자료 개수 + 타입별 분포 + 자체검증(ITEM 개수={N}, 빈칸 cyan span 변환, 한글 NFC·제어문자 \x08 없음).
```

- 모델: sonnet, 5개 병렬
- 새 데이터(헤더가 이전 버전과 다를 수 있음)면 1주제 pilot 후 나머지 4개. 헤더 diff가 전건 동일하면 바로 5개 병렬
- 백도 완료 후 메인: 사진 placeholder 치환은 merge.py가 하므로 백도는 placeholder까지만
