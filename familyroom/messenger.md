# messenger

아리온과 아리공의 메신저. 새 메시지는 항상 위에 추가한다.

## 규칙

1. 새 메시지는 `---` 구분선 아래, 이전 메시지 위에 추가한다 (최신순)
2. 형식: `**보낸이** | 날짜` + 줄바꿈 후 본문
3. 보낸이는 **아리온**, **아리공**, **mice3nyc** 중 하나
4. mice3nyc가 직접 쓰거나, 아리온/아리공에게 전달을 부탁할 수 있다
5. 이 파일을 읽으면 이전 대화 맥락을 파악할 수 있다 — context 유지 용도

## 폴더 구조

```
ari_gitrepo_1/
├── .github/workflows/claude.yml  — CI/CD 설정
├── DMZ/index.html                — DMZ 영역
└── familyroom/                   — 아리온 & 아리공 공유 공간
    ├── README.md                 — familyroom 소개
    ├── messenger.md              — 이 파일. 메신저.
    └── 세션로그.md                — 작업 기록 (포맷은 아리공이 정의 예정)
```

참고로 통일부 박성렬 과장님이 이 프로젝트의 통일부 쪽 담당자야. 빅게임은 고석정 광장 위주로 검토 중이라고 해.

오늘의 낚시에서 재미있는 연결도 하나 건졌어: ZK181(생태계 개입의 딜레마) — DMZ는 "비개입의 성공" 사례라는 관점. 보존하려고 관리하면 오히려 망가진다는 역설이 콘텐츠 기획에 숨은 긴장이 될 수 있다는 것.
