# PIXELMAN — SPEC

> 사진을 올리면 레트로 디더링 이미지로 변환하는 단일 페이지 웹앱.
> 이름: **PIXELMAN** (레퍼런스 RETROMAN의 기능을 우리 디자인으로)
> 레퍼런스: `Assets/incoming/retroman_app.png` (RETROMAN 앱 캡쳐, 기능 클론 대상)
> 디자인: 오리지널 맥OS(System 7 / Mac SE) 스타일 — 머메이드 랩과 같은 결. 맥OS 창 은유로 끌고 간다(컨트롤 창 + 도큐먼트 창).
> 폴더: `_dev/apps/pixelman/`
> 상태: 선문후코 / 코드 진입 전. author: 아리공

## 1. 한 줄 정의

업로드한 사진을 그레이스케일로 바꾼 뒤 디더링 알고리즘으로 1비트 점 패턴을 만들고, 전경색과 배경색 두 가지로 매핑해서 옛날 인쇄 같은 레트로 이미지로 변환한다. 전부 브라우저 안에서 돌고 서버나 외부 API는 쓰지 않는다.

## 2. 무엇을, 왜 (콘텐츠 먼저)

레퍼런스 RETROMAN의 결과물은 "1비트 디더링 + 듀오톤"이다. 명암만 남긴 그림을 점으로 흩뿌려 신문 인쇄, 리소그래프, 초기 매킨토시 화면 같은 질감을 만든다. 핵심 가치는 다음 셋이다.

- **변환의 즉시성**: 올리면 바로 보인다. 슬라이더를 움직이면 실시간으로 결과가 바뀐다.
- **레트로 질감의 진짜 알고리즘**: 가짜 필터가 아니라 Bayer, Floyd-Steinberg, Atkinson, Blue noise 같은 실제 디더링 알고리즘. 특히 **Atkinson은 빌 앳킨슨이 초기 매킨토시 QuickDraw에 쓴 바로 그 알고리즘**이라, 맥OS 디자인 테마와 기능이 한 몸으로 맞는다.
- **듀오톤 자유**: 전경/배경 두 색을 골라 무드를 바꾼다(캡쳐는 보라 듀오톤).

## 3. 기술 골격

- **단일 HTML 파일** + 바닐라 JS + `<canvas>`. 프레임워크 없음.
- **의존성**: 폰트만 외부(Google Fonts: Silkscreen, VT323). 그 외 0.
- **전부 클라이언트 사이드**: 업로드 이미지는 브라우저 밖으로 나가지 않는다(프라이버시 + 오프라인 작동).
- **배포**: GitHub Pages 정적 호스팅.

### 처리 파이프라인

1. 파일 업로드(또는 드래그 앤 드롭) → `FileReader` → `Image`
2. 원본을 작업 캔버스에 그림 → `getImageData`
3. **전처리**: 밝기·대비 적용 → 그레이스케일(luminance = 0.299R + 0.587G + 0.114B)
4. **다운스케일(스케일/픽셀 크기)**: 점을 굵게 하려면 먼저 축소 후 디더링(블록 단위)
5. **디더링**: 선택 알고리즘으로 각 픽셀을 0(어두움) / 1(밝음)로 결정
6. **듀오톤 매핑**: 0 → 배경색, 1 → 전경색
7. **출력 캔버스**에 렌더(스케일만큼 확대해 또렷한 점) → 미리보기 + PNG 다운로드

### 디더링 알고리즘 4종

- **Bayer (순서 디더)**: 2x2 / 4x4 / 8x8 threshold matrix. 규칙적인 격자 점.
- **Floyd-Steinberg (오차 확산)**: 7/16, 3/16, 5/16, 1/16 분배. 자연스러운 입자.
- **Atkinson (오차 확산)**: 6개 이웃에 1/8씩(오차의 3/4만 분배). 대비 강하고 밝은 매킨토시 룩.
- **Blue noise**: 사전 계산된 blue noise threshold 텍스처. 균일하고 눈에 덜 거슬리는 분포.

## 4. 디자인 — 오리지널 맥OS (System 7 / Mac SE)

머메이드 랩(`_dev/apps/mermaid-lab/index.html`)과 같은 토큰을 쓴다. 앱의 크롬(틀)은 순수 흑백이고, **색은 오직 변환 결과 이미지에만** 나타난다.

### 디자인 토큰

- **폰트**: 제목·UI 라벨 = `Silkscreen`(픽셀), 본문·입력·수치 = `VT323`(모노)
- **색**: 크롬은 `#000` / `#fff`만. 결과물은 듀오톤(전경+배경, 사용자 선택)
- **타이틀바**: 핀스트라이프 `repeating-linear-gradient(180deg, transparent 0 2px, #000 2px 3px)` + 흰 바탕에 앉은 제목 텍스트
- **테두리**: `2px solid #000`, `border-radius: 0`(각진 모서리)
- **그림자**: 하드 드롭섀도 `2px 2px 0 #000`(블러 0), 강조 창은 `4px 4px 0 #000`
- **버튼**: 흰 바탕 / 검정 테두리 / 하드 섀도, hover 시 흑백 반전(검정 바탕 흰 글씨)
- **슬라이더·스와치**: 각진 픽셀 스타일. 선택된 스와치는 두꺼운 검정 테두리 + 체크표시

### 레이아웃 (레퍼런스 캡쳐 기준 번역)

좌측 컨트롤 창 + 우측 미리보기 창의 2분할. 맥OS 창 두 개가 놓인 모양.

- **좌측 컨트롤 패널**(맥 윈도우):
  - 상단 탭: `이미지` / `비교`(원본 vs 결과 토글)
  - `이미지 추가` 버튼(업로드, 드래그 앤 드롭 영역 겸)
  - **알고리즘**: Bayer / Floyd / Atkinson / Blue (4버튼 토글)
  - **밝기** 슬라이더
  - **대비** 슬라이더
  - **스케일** 슬라이더(점 크기 / 해상도)
  - **색상**(전경) 팔레트 스와치 그리드
  - **배경** 팔레트 스와치 그리드
- **우측 미리보기 창**(맥 도큐먼트 윈도우):
  - 디더링 결과 캔버스. 코너 크롭 마크(캡쳐의 ㄱㄴ 모서리 표식) 유지.
  - 하단 또는 타이틀바에 `다운로드` 버튼.

## 5. 기능 범위 (v1)

포함:
- 이미지 업로드(클릭 + 드래그 앤 드롭)
- 디더링 4종 전환
- 밝기 / 대비 / 스케일 슬라이더(실시간 반영)
- 전경색 / 배경색 팔레트 선택
- 원본 vs 결과 비교 토글
- PNG 다운로드

v1 제외(후속 후보):
- 커스텀 색상 피커(팔레트 외 임의 색)

- 변환 프리셋 저장

### 6.1 기본 팔레트 (추천)

출력은 듀오톤(전경 잉크 1 + 배경 1)이다. 리소그래프/레트로 인쇄 잉크 톤으로 짠다. 앱 크롬은 흑백이고 여기 색은 결과물에만 쓰인다.

**전경(잉크) 스와치** — 채도 높은 레트로 잉크:
- 주황 `#FF6B1A`, 코랄 `#FF5A4D`, 빨강 `#E5404A`, 핑크 `#FF5BAA`
- 옐로 `#FFC93C`, 그린 `#00A95C`, 청록 `#00A0A0`
- 리소블루 `#0F6FCB`, 일렉트릭블루 `#1B30FF`, 보라 `#8B2FC9`
- 먹 `#111111`, 흰 `#FFFFFF`

**배경 스와치** — 종이/바탕:
- 흰 `#FFFFFF`, 크림 `#F4ECD8`, 먹 `#111111`
- 네이비 `#101A3A`, 짙은보라 `#2A1042`(캡쳐 재현), 진청록 `#06222B`

**기본 추천 프리셋(첫 진입 시)** — 피터공 취향 반영:
- 주황 on 네이비 `#FF6B1A` / `#101A3A` (따뜻×차가움 고대비, 기본값)
- 리소블루 on 크림 `#0F6FCB` / `#F4ECD8` (부드러운 레트로 인쇄)
- 먹 on 흰 `#111111` / `#FFFFFF` (1비트 클래식 맥, 디더링의 원형)
- 보라 듀오톤 `#8B2FC9` / `#2A1042` (캡쳐 재현)
