## todoy-bar — 개발 계획 (live)

> 이 노트가 todoy-bar 엔트리 포인트. 돌아오면 이것부터 읽는다. 항상 "지금 상태".

### 목적·의도

피터공이 하루를 시작할 때 떠오르는 오늘 할 일을 짧은 목록으로 메뉴바에 올려두고, 지금 하는 것을 ACTIVE로 골라 흘끗 보고, 완료를 체크한다. now-bar(현재형, 지금 무슨 작업)·Context Hop Bar(회고형, 어제까지)와 합쳐 시간축 삼각형의 **계획축(오늘)**을 맡는다.

### 설계 방향

- **now-bar 패턴 재활용**: SwiftBar 플러그인 + 헬퍼 스크립트 + 파일 데이터. 권한은 헬퍼 절대경로만 allowlist.
- **단순함 우선**: DN·now-bar와 독립(피터공 결정). 동기화 얽힘 없음.
- **메뉴바 완결**: 표시뿐 아니라 클릭으로 ACTIVE 선택·완료 토글, osascript 입력창으로 항목 추가까지 메뉴바 안에서.
- **굿모닝 세팅 / 굿바이 이월**: 기존 루틴에 얹는다.

### 현재 상태

- 2026-06-02: 착수 → SPEC → 헬퍼/플러그인 구현 → 헤드리스 검증 → **라이브(메뉴바 표시 확인)**. 인터랙션 정정(목록 클릭=ACTIVE, 서브=완료 ☑, 체크박스 ☐·☑). now-bar는 피터공 "필요없다"로 `_archive/`에 폐기, todoy 단일화(SwiftBar PluginDirectory도 todoy-bar로). **다음 = 굿모닝/굿바이 스킬 연동(`setup` 이월 / `carryover` 미완료).**
