# Gallery 98 (gallery98.org) 사이트 스타일 분석

> 분석일: 2026-03-24
> 목적: 유사 스타일 사이트 개발 참고용

---

## 1. 사이트 개요

- **이름**: Gallery 98
- **URL**: https://gallery98.org
- **설립**: 2014년, Marc H. Miller (아티스트/미술사학자/큐레이터)
- **콘텐츠**: 1960s~2000s 아트 에페메라 (갤러리 안내카드, 포스터, 출판물 등)
- **목적**: 컬렉터 & 연구자를 위한 온라인 갤러리/카탈로그/스토어
- **소장품**: 8,700+ 아이템, 1,000+ 아티스트

## 2. 디자인 스타일 요약

| 항목 | 설명 |
|------|------|
| **전체 톤** | 에디토리얼 / 아카이브 / 학술적 |
| **디자인 방향** | 콘텐츠 중심, 미니멀하지만 정보 밀도 높음 |
| **느낌** | 미술관 도록 + 온라인 카탈로그 |

## 3. 레이아웃 구조

### 플랫폼
- **CMS**: WordPress 기반 (URL 구조로 확인: `/category/`, `/news/` 패턴)
- **커머스**: 온라인 스토어 기능 포함 (판매/판매완료 표시)

### 페이지 구조
```
┌─────────────────────────────────┐
│  헤더: Gallery 98 로고/타이틀     │
├─────────────────────────────────┤
│  네비게이션 메뉴                  │
│  (About, Artists, Ephemera,     │
│   Collections, Newsletter...)   │
├─────────────────────────────────┤
│                                 │
│  메인 콘텐츠 영역                │
│  - 블로그/카탈로그 형태           │
│  - 카테고리별 아이템 나열         │
│  - 이미지 + 서술형 설명           │
│                                 │
├─────────────────────────────────┤
│  푸터                           │
└─────────────────────────────────┘
```

### 주요 섹션들
- **Artists 페이지**: 알파벳순 아티스트 목록 → 개별 아티스트 관련 아이템
- **Ephemera 페이지**: 32개 인기 테마 카테고리 스포트라이트
- **Collections**: 갤러리, 나이트클럽, 퍼포머, 미디어, 미술운동, 정치 등 주제별
- **개별 아이템**: 이미지 + 상세 서술 (역사적 맥락, 아티스트 정보)

## 4. 카테고리 체계

사이트의 강점은 **다양한 분류 체계**:
- 아티스트별 (1,000+)
- 갤러리별 (Downtown Galleries, East Village/LES Galleries, MoMA 등)
- 주제별 (Architecture, Furniture & Design, Photography 등)
- 지역별 (New York City 등)
- 미디어별

## 5. 콘텐츠 전략

| 특징 | 설명 |
|------|------|
| **에디토리얼 스토리텔링** | 각 아이템에 풍부한 역사적 맥락 설명 |
| **연구자 친화** | 판매완료 아이템도 삭제하지 않음 (학술 참고용) |
| **"판매완료 숨기기"** | 컬렉터용 필터 옵션 제공 |
| **뉴스레터** | 새 아이템/이야기 발행 |
| **교차 참조** | 아이템 간 연결 (아티스트↔갤러리↔시대↔운동) |

## 6. 이런 사이트를 만들려면

### 기술 스택 추천

```
옵션 A: WordPress (원본과 동일한 접근)
─────────────────────────────────────
- WordPress + WooCommerce (또는 커스텀 포스트 타입)
- 갤러리/포트폴리오 테마 (깔끔한 에디토리얼 스타일)
- Advanced Custom Fields (커스텀 메타데이터)
- 카테고리/태그 시스템 활용

옵션 B: 정적 사이트 (단순 + 빠름)
─────────────────────────────────────
- Hugo / Astro / 11ty
- Markdown 기반 콘텐츠
- 카테고리 분류 시스템
- 이미지 최적화 파이프라인

옵션 C: 바닐라 HTML/JS (현재 팀 역량과 일치)
─────────────────────────────────────
- 단일 또는 소수 HTML 파일
- JSON/CSV 기반 아이템 데이터
- 바닐라 JS로 필터링/검색
- GitHub Pages 배포
```

### 핵심 기능 구현 포인트

1. **카탈로그 시스템**
   - 아이템별 이미지 + 제목 + 설명 + 메타데이터
   - 다중 카테고리 분류 (아티스트, 시대, 장르 등)
   - 검색 & 필터링

2. **에디토리얼 레이아웃**
   - 이미지 좌/우 배치 + 텍스트 서술
   - 충분한 여백, 읽기 편한 줄 너비 (45~75자)
   - 서체: 세리프 or 클래식 산세리프

3. **아카이브 느낌의 디자인**
   - 흰 배경 또는 약간 따뜻한 배경 (아이보리)
   - 블랙/다크그레이 텍스트
   - 최소한의 UI 장식
   - 이미지가 주인공

4. **네비게이션**
   - 카테고리 기반 브라우징
   - 알파벳순 아티스트 인덱스
   - 테마별 스포트라이트 섹션

### 디자인 참고 키워드
- Editorial web design
- Archive/catalog website
- Art gallery online catalog
- Museum collection browser
- Content-heavy minimal layout
