# Being Faust Values — 가치 카드 선택 웹앱 SPEC

> **버전**: v0.1 (초안)
> **작성**: 아리공, 2026-04-06
> **상태**: 피터공 검토 대기

---

## 1. 한 줄 요약

워크숍/특강 참가자가 **개인 모바일**로 12개 가치 카드 중 6개를 선택하고 우선순위를 정하면, 진행자가 **PC 화면**에서 취합된 결과를 실시간으로 보며 이야기를 나누는 도구.

---

## 2. 왜 만드는가

빙파우스트 원작의 핵심 경험 — "당신이 욕망하는 가치는 무엇인가?" — 을 워크숍 세팅에서 가볍게 재현.
원래 앱은 2014년 제작, 현재 서버 없음. 이 웹앱은 가치 선택 부분만 추출해서 독립적으로 쓸 수 있게 한다.

## 4. 사용자 플로우

### 4A. 플레이어 (모바일)

```
[1. Welcome] → [2. Card Selection] → [3. Confirm] → [4. Priority] → [5. Done]
```

#### Screen 1: Welcome
- MEPHISTO & co. 로고 + 분위기 사진 (배경)
- "Mephisto & Co.에 오신 것을 환영합니다"
- 입력: 이름 (필수), 방 코드 (4자리)
- [입장하기] 버튼

#### Screen 2: Card Selection (6/12 선택)
- 12개 카드가 3x4 그리드로 표시 (카드 앞면 이미지 사용)
- 탭하면 선택 토글 (선택됨 = 테두리 하이라이트 + 체크)
- 상단에 "6개를 선택하세요" + 카운터 (3/6)
- 6개 선택 완료 시 [다음] 버튼 활성화

#### Screen 3: Confirm Selection
- 선택한 6개 카드를 보여줌
- "당신이 선택한 가치입니다"
- [다시 선택하기] — Screen 2로 복귀
- [다음: 순위 정하기] — Screen 4로

#### Screen 4: Priority Ranking
- 선택한 6개 카드를 드래그앤드롭으로 순위 정렬
- 1st ~ 6th 라벨 표시
- 모바일 터치 드래그 지원 필수
- [제출하기] 버튼

#### Screen 5: Done
- "당신의 가치가 기록되었습니다"
- 자신의 선택 요약 표시
- 대기 화면 (진행자가 결과를 공유할 때까지)

### 4B. 진행자 (PC 브라우저)

```
[1. Create Session] → [2. Lobby] → [3. Live Dashboard]
```

#### Screen 1: Create Session
- Being Faust 로고
- [새 세션 만들기] → 4자리 방 코드 자동 생성
- QR 코드 표시 (참가자 접속용)
- 참가자 접속 URL 표시

#### Screen 2: Lobby
- 접속한 참가자 이름 실시간 표시
- 참가자 수 카운터
- [결과 보기] — 제출 완료된 참가자가 있으면 활성화

#### Screen 3: Live Dashboard (핵심 화면)
- **참가 현황**: N명 참가 / M명 제출 완료
- **가치별 선택 빈도**: 12개 가치의 수평 막대 차트 (선택 횟수)
- **우선순위 가중 랭킹**: 1위=6점, 2위=5점, ... 6위=1점으로 가중합
  - 1등 가치 Top 3 (가장 많이 1위로 꼽힌)
  - 2등 가치 Top 3
  - 3등 가치 Top 3
- **선택되지 않은 가치**: 아무도 고르지 않은 가치가 있다면 별도 표시
- 실시간 업데이트 (새 제출이 들어올 때마다 차트 갱신)

## 5. 기술 아키텍처

### 원칙
- **단일 HTML 파일** (player.html, dashboard.html 각 1개)
- **바닐라 JS** — 프레임워크 없음
- **실시간 동기화** 필요 — 참가자 제출 → 진행자 화면 즉시 반영

### 백엔드: Google Sheets (확정)

- **Google Sheets**를 DB로 사용
- **Google Apps Script**로 Web App 배포 → REST API 역할
- 참가자 제출 → Apps Script doPost → Sheets에 한 행 추가
- 대시보드에서 doGet으로 전체 데이터 조회 → 차트 렌더링
- **장점**: 기존 구글 계정 사용, 서버 관리 없음, 결과 스프레드시트로 재활용 가능
- **단점**: 1~3초 지연 (실시간은 아님, 새로고침 방식), Apps Script 일일 호출 제한 있으나 워크숍 규모에 충분
- **비용**: 무료

### 호스팅
- GitHub Pages (기존 `_dev/` 리포 활용)
- URL: `https://mice3nyc.github.io/ari_gitrepo_1/beingfaust_values/`
- 또는 별도 도메인 연결 가능

### 파일 구조 (예상)
```
beingfaust_values/
├── SPEC.md            ← 이 파일
├── index.html         ← 플레이어용 모바일 웹앱
├── dashboard.html     ← 진행자용 PC 대시보드
├── imgs/              ← 로고
├── valuecards/        ← 12개 카드 이미지 (앞면 + 뒷면)
└── references/        ← 디자인 레퍼런스 (배포 안 함)
```

## 6. 디자인 시스템

### 톤
원작 앱의 무드를 계승:
- **다크 테마** — 배경 #1A1A1A ~ #2D2D2D (메피스토의 상점)
- **서체** — 깔끔한 산세리프 (시스템 폰트)
- **MEPHISTO & co.** — 원작 로고 폰트 (세리프, 특수 Ø)
- **액센트**: 각 가치 카드의 고유 컬러 활용
- **버튼**: 코랄/레드 계열 (#E74C3C) — 원작 START GAME 버튼 색상 계승

### 카드 표현
- 모바일 그리드: 3열 × 4행
- 카드 비율: 원본 이미지 비율 유지 (약 2:3)
- 선택 상태: 카드 테두리 하이라이트 (해당 가치의 액센트 컬러)
- 미선택: 약간 어둡게 (opacity 0.7)

### 반응형
- **플레이어**: 모바일 최적화 (375px 기준), 세로 고정
- **대시보드**: 데스크탑 최적화 (1280px+), 프로젝터/대형 모니터 고려
