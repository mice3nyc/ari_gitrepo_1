# PLAN — TM (TerminalMonitor)

## 목적

멀티 창(클로드코드 인스턴스) 동시 작업에서 "어느 창이 뭘 하는지" 혼돈을 없앤다.
메뉴바에서 흘끗 보고 각 창의 현재 작업·당일 로그를 파악. todoy-bar(계획축)의 실행축 짝.

## 설계 방향

- **SwiftBar 확장** — todoy-bar 패턴 재사용(플러그인 + tm.sh 헬퍼 + 날짜/창별 JSON + 스킬 연동). 별도 메뉴바 아이콘.
- **로테이션 표시** — 메뉴바 자리에서 A→B→C→D 순환(3초), 새 업데이트 창 핀 우선. SwiftBar 새로고침 주기 = 로테이션 주기.
- **약한 고리 해소** — now-bar가 죽은 이유("손으로 적어야 산다")를 CLAUDE.md 행동 룰로 강제. 단 퍼포먼스 안전선(작업 단위 갱신, 매 툴콜 X).
- **스킬 통합** — recall/goodmorning에서 창 등록 질문, goodbye에서 `TM_log_YYMMDD` 노트로 로그 통합 + DN 링크.
