### 이 프로젝트가 무엇인가

MOMAK은 2013년 런칭한 놀공의 원조 빅히트 빅게임. 10~15명이 한 팀이 되어 커다란 보드에서 10년(10턴) 규모의 사업을 운영·투자·확장하는 현장형 멀티플레이어 빅게임이다. 삼성전자 신입교육에서 태어나 삼성물산·삼성화재·KB손해보험·제일기획·미래에셋·독일문화원 등으로 판을 넓혔고 누적 참가자가 만 명을 넘는다. 2026 재개발은 **신기능의 발명이 아니라, 14년간 검증된 구조를 현대 웹 스택으로 재건**하는 일이다 (2014·2016 설계 엑셀에 실시간 시뮬레이션 수치가 이미 다 있음).

### 한 턴의 구조 — 마켓타임 + 보드타임

```
1턴(=1년) = Market Time (최소 10분, 돈 버는 시간) + Check Time (보드 운영)
             └ 이번 개발 범위 ──┘              └ 후속 범위 ─┘
```

- **Market Time**: 마켓(주식시장식)이 열려 팀원들이 담당 아이템을 사고팔아(BUY/SELL) 시세차익으로 운영자금을 불린다. (원게임은 이어서 교환소에서 아이템→사업칩 교환.)
- **Check Time**: 마켓이 닫히고 교환한 사업칩을 보드에 배치해 사업을 완수하고 매출을 낸다.

### 개발 범위 — 마켓타임 먼저 (v1)

이번에 만드는 것은 **Market Time만**이다. 그 안에서도 1차 범위를 좁혔다:

| 포함 (v1) | 제외 (후속) |
|---|---|
| 플레이어 BUY/SELL + 보유 수량 확인 | TRADE(주식→칩 교환)·교환소·교환원 |
| 실시간 가격 진동·동기화 (server-authoritative) | 물리 사업칩·보드 배치 |
| 마켓 티커 디스플레이(태블릿 1~2아이템) | 사업 완수·매출 정산·distribute |
| 중앙 모니터링(거래 지표) | 회고·투표·랭킹 그래프 |
| GM 콘솔(마켓 개폐·정지·자금 지급·턴 조정) | 신사업·VALUE UP·WARNING 이벤트 |

> 교환·정산은 보드타임 영역이라 v1에서 뺀다. 단 데이터 모델에는 확장 훅(`exchangeEnabled` 등)을 남겨 나중에 붙이기 쉽게 한다.

### 확정된 결정 (2026-07-11 피터공)

- **A1 계보**: **미래에셋 에디션**(11턴 = 튜토리얼 1 + 진행 10) 하나로 시작. 이유 = 아이템 16종 + 턴별 min/delta 가격 데이터가 가장 완성됨. 데이터 원본 → `docs/data/market_config_miraeasset.json`.
- **A2 TRADE**: v1 제외. BUY/SELL + 수량 확인만.
- **A3 정산**: v1 제외(보드타임). 중앙 DISPLAY는 거래 지표(운영자금·주식소유·구매/매각량)까지만.
- **A4 가격 진동 (2026-07-11 재설계 — 톱니 사이클)**: `min_price`=저점, `delta_price`=스텝 상승폭. 저점→delta씩 상승→고점(=다음 턴 저점) 찍으면 저점으로 급락→반복. **예측 가능한 사이클**(랜덤 워크 폐기 — 예측 불가면 매매 타이밍이 안 생김). 아이템마다 속도·폭 다름, 턴 시작 위상 랜덤. 튜닝 노브(가격변동시간·속도·폭 배수). 이벤트층(폭락/급등)은 다음. 상세 = [[SPEC-market]] §4.
- **B5 플랫폼**: 전부 브라우저 웹앱(플레이어 폰·마켓 태블릿·GM콘솔·중앙디스플레이 모두 URL 접속). 프런트 = `_devhaus/momak2026-front`(React+Vite), 백 = 최소 웹소켓 서버 신설(`momak2026-back`).
- **B6 세팅**: 엑셀/CSV import 우선(데이터가 이미 스프레드시트에 있음) + GM 화면 편집.
- **B7 규모**: 목표 = 단일 세션 최대 ~120명(도면 최대치 4~7팀 × 14~15명). 첫 부하는 그보다 작게 시작해 키움.
- **B8 디자인**: 2013 마켓 화면 톤(다크 배경 + 상승/하락 색 박스) 계승. 남은 목업(MARKET_MARKETSCREEN·toray-28·중앙디스플레이)이 기준.

### 기술 방향

- **server-authoritative 실시간**: 서버가 시간·가격·잔액·재고의 유일한 원장. 모든 화면이 같은 세션 상태·가격 버전을 구독. "보고 있는 가격으로 체결"과 광클릭 안전성이 핵심 요구.
- **덱스(codex) 스펙 차용**: `Assets/incoming/momak2026/codex_spec/`의 외부 초안에서 상태기계·Quote 체결정책·멱등 원장·오류코드·인수조건을 차용. 단 (범위=교환 빼기 / 인프라=최소부터 / 데이터=실값 주입) 3축은 우리 것으로 교체. 검토 = [[요청.26.0711.1950-MOMAK마켓타임]] 부록 B.
- **인프라**: 놀공 AWS. 덱스의 풀스택(ECS/Aurora/ElastiCache)은 **목표 상태(북극성)**로 두고, 첫 빌드는 최소 웹소켓 서버(방 생성 + 멀티 브라우저 실시간 동기화)부터 피터공+아리공 합동 세팅.

### 기기 구성 (v1에서 만드는 화면)

1. **Player App** — 플레이어 개인 폰. 로그인 → 담당 아이템별 BUY/SELL + 보유수량·운영자금.
2. **Market Channel Display** — 태블릿. 1~2개 아이템의 실시간 가격 티커.
3. **Market Overview Display** — 중앙 대형 스크린. 전체 아이템 가격 + 팀 거래 지표 모니터링.
4. **GM Console** — 진행자. 세션·턴·마켓 상태, 자금 지급, 가격/타이머 제어.
5. **(후속) Game Studio** — 게임 버전·설정 제작. v1은 엑셀 import + 최소 편집으로 대체.
