# DMZ 다이어리 — UI 맵

피터공↔아리공↔아리온↔백도 에이전트가 같은 언어로 소통하기 위한 네이밍 규칙.

## 화면 (Screen)

| 이름          | 코드 ID              | 설명                                  |
| ----------- | ------------------ | ----------------------------------- |
| **로그인**     | `#login-screen`    | 타이틀 + 이름/코드 입력                      |
| **선택**      | `#select-screen`   | 캐릭터(시선) 선택                          |
| **게임**      | `#game-screen`     | 아래 두 상태를 포함하는 하나의 screen            |
| ↳ **게임-목록** | `.game-content`    | 자료카드들이 나열된 상태                       |
| ↳ **게임-뷰어** | `.source-detail`   | 카드 클릭 후 전체화면 오버레이 (position: fixed) |
| **전달**      | `#delivery-screen` | 두루미 애니메이션 + 기록 전송                   |
| **결과**      | `#result-screen`   | 점수 + 요약                             |

## 기능 명세

### 자료 네비게이션 (게임-뷰어 좌우 전환)

**목적**: 게임-뷰어에서 게임-목록으로 돌아가지 않고 자료 간 직접 이동

**동작**:
- 게임-뷰어 화면 좌우 끝(약 44px)에 투명한 탭 영역
- 좌측 클릭 → 이전 자료 (A→D→C→B→A 순환)
- 우측 클릭 → 다음 자료 (A→B→C→D→A 순환)
- 터치 스와이프도 동일하게 동작
- hover 시 < > 화살표 표시 (미니멀)

**비활성화 조건**:
- 답변모달(`.answer-modal`)이 열려 있을 때
- source-detail이 비활성일 때

**상태 저장**: `state.currentSourceId`에 현재 열린 자료 ID 저장

**관련 함수**: `openSource()`, `closeSource()`, `navigateSource(direction)`

### 캐치워드 (텍스트 하이라이트)

**목적**: 자료 읽기 중 핵심 단어를 마킹하여 기억 보조

**동작**:
- 게임-뷰어 본문 텍스트 클릭 → 클릭된 단어를 형광연두 하이라이트
- 클릭한 자료의 클릭한 위치만 하이라이트 (교차 하이라이트 없음)
- 최대 3개 유지 (전체 자료 통틀어서). 4번째 클릭 시 가장 오래된 것 해제 (FIFO, 어느 자료에 있든)
- 이미 하이라이트된 단어 재클릭 → 해제 (토글)
- 빈칸(`.blank-slot.empty`) 클릭은 캐치워드 대상 아님 → 기존 답변모달 동작
- 복원된 빈칸(`.blank-slot.filled`) 클릭 → 안의 텍스트 전체를 하나의 캐치워드로 처리 (공백 포함해도 통째로)

**단어 경계 규칙**:
- 공백(스페이스, 줄바꿈, 전각공백)만으로 단어 구분
- 쉼표, 마침표, 괄호 등 구두점은 단어에 포함 (예: "1,292개" → 통째로, "DMZ." → 통째로)

**상태 유지**:
- `state.catchwords = [{ sourceId, wordIndex, text }, ...]` (최대 3개)
- 자료 간 이동(좌우 네비)해도 유지
- 자료를 열 때 현재 sourceId의 캐치워드만 하이라이트 적용

**초기화**: 새 캐릭터 선택 시 리셋 (향후 수동 리셋/지우개 버튼 추가 가능)

**스타일**: 형광연두 (`#ccff00` 계열), `.catchword-highlight`

**충돌 방지**: 스와이프 네비게이션과 텍스트 탭 구분 필요 — 스와이프(이동)는 네비, 탭(정지 클릭)은 캐치워드

**향후 확장 가능성**:
- 하이라이트된 단어 목록을 화면 어딘가에 표시
- 수동 리셋 버튼 (지우개)
- 캐릭터 전환 후에도 기록 유지 옵션

**관련 함수**: `applyCatchwords()`, `buildWordIndex()`, `getWordAtClick()`, `getWordIndexAtClick()`, `getFilledBlankIndex()`

- "**답변모달**의 버튼이 작아"
- "**편지** 자료의 **빈칸** 색상 변경"
