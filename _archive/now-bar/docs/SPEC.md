## Now Bar — SPEC (v3)

> 선문후코 산출물. 코드 작성 전 명세. 변경은 이 문서를 먼저 갱신 후 코드.
> **전환 이력**: v1 큰 시계(실패: "그냥 시계다") → v2 tool 로그 피드(실패: 기능 나열이라 무슨 작업인지 안 보임) → **v3 아리공 작업 내러티브**.

### 한 줄 정의

아리공이 "지금 무슨 작업을 하는 중"이라고 적은 한 줄을, 시작 이후 경과와 함께 크게 보여주고 그 아래 오늘 작업 흐름을 쌓는 창. 흘끗 보면 "지금 아리공이 뭘 하는지"가 인간 말로 읽힌다. 데이터는 도구 호출(hop 로그)이 아니라 작업 의도다.

### 왜 이 전환인가 (피터공 피드백 2026-06-02)

"기능을 쓰면 내가 뭘 하는지 어떻게 알아." `편집 index.html`은 든 연장이지 만드는 작업이 아니다. 망치질을 백 번 봐도 집을 짓는지 책장을 짜는지 모른다. 의미는 도구가 아니라 의도에서 오고, 의도는 아리공이 적어야 로그에 들어온다.

### 데이터 소스

- **`now_status.jsonl`** (now-bar 폴더, 아리공이 직접 적음). 한 줄 = 한 작업 단위.
  - `{"ts":"2026-06-02T11:06:26","headline":"한 줄 작업명","detail":"보조 한 줄(선택)","tag":"프로젝트(선택)"}`
- **아리공 운영 규칙**: 작업을 시작하거나 전환할 때 headline 한 줄을 append. 굵직한 단위(요청 단위, 주제 전환)마다. 매 tool마다가 아님 — 너무 잘게 쪼개지 않는다.
- hop 로그(context-hop-bar)는 v3에서 데이터로 쓰지 않는다. (후속 옵션: 맨 아래 raw 보조 줄)

### 표시

- **머리글 (작게)**: 현재 시각 `HH:MM:SS` + 날짜·요일. 더는 주인공 아님.
- **지금 하는 일 (주인공, 큰 글씨)**: 최신 status의 `headline` + `detail`(작게) + `N분째`(최신 ts부터 경과).
- **그 전까지 한 일**: 이전 status들 최신순, `HH:MM  headline` 한 줄씩(옅게).
- **빈 status**: "대기 중" 표시, 시각은 항상 뜸.

### 갱신 주기

- 시각·경과: `setInterval` 1초.
- status 재fetch: 8초.

### 디자인

- **폰트**: 시스템 폰트 `-apple-system, BlinkMacSystemFont, "Apple SD Gothic Neo", "Pretendard", sans-serif`. 한글 가독 우선. 도스 비트맵(HBIOS) 폐기.
- **톤**: 흰 배경, 검정, 점선 프레임 미니멀. 지금 하는 일만 또렷, 지난 흐름은 옅게.
- **창**: 가로 넓은 슬림. (`--app` 창이 불안정 → 당분간 일반 Chrome 창/탭 허용. app 창 안정화는 후속.)

### 실행

- `start_server.sh` 그대로(포트 8788). `now_status.jsonl`은 서버 루트(now-bar 폴더)라 바로 fetch됨. logs 심링크는 v3에선 미사용(보존).

### 검증 기준

1. 최신 status의 headline이 크게, 경과(N분째)가 함께 뜬다.
2. status를 한 줄 append하면 8초 내 반영되고, 직전 게 "그 전까지 한 일"로 내려간다.
3. 빈 status / 이른 아침에 안 깨짐(시각은 항상).
4. 한글이 또렷(시스템 폰트).
5. 시각 1초 tick 정확.
