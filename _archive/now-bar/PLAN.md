## Now Bar — 개발 계획 (live)

> 이 노트가 now-bar 엔트리 포인트. 돌아오면 이것부터 읽는다. 항상 "지금 상태".

### 목적·의도

피터공이 멀티태스킹 중 "지금 내가 뭐 하던 중이었지 / 시간이 어떻게 됐지"를 놓칠 때, 흘끗 보면 맥락이 복구되는 작은 현재형 상태창. Context Hop Bar(회고형)의 짝 = 현재형(now).

### 설계 방향

- **데이터는 만들지 않고 빌린다**: Context Hop Bar의 hop 로그(JSONL)를 공유해서 읽기 전용 뷰만 새로. 로거·파이프라인 재사용.
- **시간이 주인공**: 현재 시각 + 오늘 누적이 중심(피터공 선택). "지금 무엇을"은 보조 한 줄.
- **계기판 미감**: HBIOS-SYS 도스 시스템 폰트. 흰 배경 + 검정 모노 + 점선 프레임으로 미니멀 절충.
- **미니멀 v1 먼저**: 1개 체험 가능한 형태로 띄우고 피터공 반응 받아 확장(타임라인 띠 등은 후속).

### 현재 상태

- 2026-06-02: 착수 → v1(시계)·v2(tool 피드) 둘 다 피터공 반려 → **v3 라이브**. 핵심 전환 = 데이터를 "시스템이 뭘 호출했나(hop 로그)"에서 "아리공이 무슨 작업을 한다고 적었나(now_status.jsonl)"로. 시스템 폰트. v3 = 머리글(시각 작게) + 지금 하는 일(큰 headline+detail+경과) + 그 전까지 흐름. headless 검증 통과, 한글 또렷·내러티브 읽힘. **운영의 핵심: now-bar가 살아있으려면 아리공이 작업 전환마다 now_status.jsonl에 한 줄을 적어야 한다(지금 수동).** **다음 = 피터공 육안 피드백 → status append 습관/자동화 정착.**
