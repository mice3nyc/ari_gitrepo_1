## SPEC — 화면 명세 (1단계)

> 기준: [[SPEC-mechanics]]. react-router. 화면은 store만 호출. 디자인: 흰 배경 B&W 미니멀, 점선·얇은 선(미술관 톤). 카드 텍스트가 주인공.

### 라우트

`/start` → `/select` → `/play` → `/result`. (manifestia App.jsx 가드 패턴: 게임 미시작이면 /play·/result 접근 시 /start로.)

### 1. `/start` 입장

- 타이틀 + 한 줄 소개("내 머릿속 충동을 들여다보고, 생각으로 다스리는 게임") + 시작 버튼.
- 진행 중 저장 있으면 "이어하기 / 새로 시작" 분기. store `getGameState().phase` 확인.

### 2. `/select` 충동 선택

- "내가 경험한 충동을 골라보세요" — 충동카드 후보(18장, 유형 미공개)에서 **7장 선택**.
- 카드엔 충동 내용만 보임(유형 라벨 가림 — 캐넌). 선택 시 강조. 7장 채우면 다음 활성.
- 완료 → `store.selectImpulses(ids)` → `/play`.

### 3. `/play` 하루 루프 (핵심 화면)

좌: 게임보드(6칸×3영역) / 우: 손패·자원·액션. 한 화면에서 하루가 흐름.

- **상단**: Day n · 생각하는힘(코인) · 진행(실천 n/6, 과몰입 n/6 — 캐넌 별 6개/하트바 톤).
- **드로우 단계**: "카드 뽑기" 버튼(1장씩) + "그만 뽑기" 버튼. 뽑힌 카드 손패에 쌓임(최대치 가드).
- **구매(억제) 단계**: 손패 생각하는힘으로 생각카드 구매 → 같은 유형 손패 충동 억제(보드 생각칸 누적·충동 버린덱). `store.buyThought`.
- **하루 정산**: "하루 마치기" → `store.resolveDay`(억제 실패 충동 과몰입 칸 적립 + 발동 효과) → `store.formActions`(같은 유형 생각 2장 → 실천칸) → `DayResult` 모달(이 유형 충동을 느꼈고, 이 생각으로 억제했고, 못 막은 건 이런 과몰입으로 이어졌다 — 캐넌의 "객관적으로 바라보기").
- `store.checkEnd` → cleared/failed면 `/result`.

> 인터랙션 캐넌: 유형 1:1 매칭(같은 유형이라야 억제) · 유형 미공개로 내용 읽게 · 구매 행동으로 실천/생각 텍스트 읽게.

### 4. `/result` 결과

- 클리어("6가지 충동을 모두 다스렸습니다") / 실패("과몰입이 가득 찼습니다 — 돌이킬 수 없는 상태"). 캐넌: 실패 순간이 감정적으로 강렬해야 치료적.
- 며칠(턴) 걸렸는지 · 다시하기. (누적 기록 화면은 2단계.)

### 컴포넌트

- `Card` — 충동/생각/실천/과몰입 공용. type prop으로 점선 테두리·라벨 처리(충동은 유형 가림).
- `GameBoard` — 6유형 행 × (과몰입/실천/생각) 영역. 캐넌 사진 레이아웃(p.10·11·15·16·30·35) 참고: 세로 칸 + 좌 사용더미/우 구매.
- `Hand` — 손패(뽑은 충동·생각하는힘).
- `ResourceBar` — 생각하는힘·진행 게이지.
- `DayResult` — 하루 정산 모달.
- `NavBar` — manifestia 복제.

### 디자인 토큰

흰 배경(#fff), 텍스트 먹색, 카드 점선 테두리(hairline), 유형별 옅은 톤 구분(채도 낮게), 강조는 단색 1개. 캐넌 페이퍼는 충동=빨강/생각=초록/실천=파랑으로 구분했으나(p.10 효과적이라는 피드백), 디지털은 B&W 기조에서 최소 색 강조로 변환 — 검토 포인트.
