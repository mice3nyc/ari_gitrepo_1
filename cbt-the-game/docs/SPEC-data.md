## SPEC — 카드 데이터 (1단계)

> 카드 콘텐츠 소스 **확정**(2026-06-22 피터공): `Assets/incoming/CBT Game/10_21 덱빌딩게임 콘텐츠.xlsx`의 **"시트1의 사본"** (K-IGD 6유형). 추출·정리 완료 → `data/cards-session1.json`.
> 다른 두 시트는 미채택: `9_10`(9유형+레벨), `241223 정예공`(교육정보 4카테고리, 유형 폐기 방향 — "유형화 존치" 미결의 흔적). 정예공 250109 교육정보는 추후 생각카드 콘텐츠 보강 참조용.

### 실데이터 현황 (추출 완료)

- 6유형 각각 **충동 3 / 과몰입 3 / 생각 3 / 실천 3** (유형당 3장 1:1 짝). 총 충동18·과몰입18·생각18·실천18.
- 유형별 **과몰입행동 발동 효과**(`overuseEffect`, 일반/강화 2티어) 존재 — 두 계열: ⓐ현저성·내성·갈등(카드뽑기 수·생각칸 축소) ⓑ기분변화·금단·재발(즉시 과몰입·억제불가·생각카드 폐기).
- 백도가 우려한 "생각카드 유형당 1장" 결손은 **실재 아님** — 충동마다 짝 생각이 있어 유형당 3장.
- `_x0008_` 백스페이스 오염 제거 완료([[볼트 백스페이스 노이즈]]).
- ⚠️ 드로우 수치 불일치: 캐넌 docx "하루 최대 5장" vs 10_21 시트 "4장 시작·최대 7장(강화 6/8)". SPEC-mechanics §2 검토 포인트로 이관.

### 1. 유형 (K-IGD 6) `[캐넌]`

`salience`(현저성) · `mood`(기분변화) · `tolerance`(내성) · `withdrawal`(금단) · `conflict`(갈등) · `relapse`(재발).
생각카드 유형 = 충동카드 유형과 동일(같은 유형 1:1 매칭). 실천카드 유형 = 과몰입행동 유형과 동일.

### 2. JSON 스키마 (`client/data/cards-session1.json`)

```jsonc
{
  "schema": "cbt-deckbuild-s1-1",
  "session": 1,
  "impulseTypes": ["salience","mood","tolerance","withdrawal","conflict","relapse"],
  "typeLabels": { "salience":"현저성", "mood":"기분변화", "tolerance":"내성",
                  "withdrawal":"금단", "conflict":"갈등", "relapse":"재발" },
  "impulses": [ { "id":"imp_001", "type":"salience", "text":"…" } ],   // 충동 선택지 (≥ 충동 14종 권장)
  "thoughts": [ { "id":"thg_001", "type":"salience", "text":"…", "cost":2 } ], // 생각카드 (구매 대상)
  "actions":  [ { "id":"act_001", "type":"salience", "text":"…" } ],   // 실천카드 (같은 유형 생각 2장→등장)
  "overuse":  [ { "id":"ovr_001", "type":"salience", "text":"…", "effect":"…" } ] // 과몰입행동
}
```

규칙:
- `id`는 종류 접두어(imp/thg/act/ovr) + 일련번호. 유형 매칭은 `type` 일치로 판정(별도 정답표 불필요).
- 같은 유형이라도 카드 내용은 서로 다르게(중복 텍스트 금지) — 캐넌 7차 "구매 때마다 새 카드를 읽게". `[캐넌]`
- 충동카드는 **유형 미공개**로 화면에 뜸(내용만 보고 유추) — `type`은 데이터에 있되 UI에서 가림. `[캐넌]`

### 3. 피터공 시트 → JSON 매핑 절차

시트 도착 시:
1. 시트의 컬럼 헤더 확인(예상: 종류 / 유형 / 내용 / 효과 / 비용 등). 실제 헤더에 맞춰 매핑표 1줄 합의.
2. `client/data/_source/` 에 원본(xlsx/csv) 보관(삭제 금지).
3. 변환 스크립트 `client/scripts/build_cards.py`(또는 노드)로 위 스키마 JSON 생성. 유형명은 한글↔key 매핑(`typeLabels` 역).
4. 검증: 6유형 각각 충동·생각·실천·과몰입이 최소 1장 이상 존재(없으면 빌드 경고). 같은 유형 생각 ≥2(실천 전환 가능해야 함).

### 4. 미정·확인 (피터공)

- 시트 포맷(xlsx/csv/구글시트 링크)과 컬럼 구성.
- 카드 수량: 유형별 충동/생각/실천 몇 장씩인지(덱 밸런스 직결).
- 생각카드 비용·레벨 존재 여부(SPEC-mechanics §2 코인 경제와 정합).
- 과몰입행동 카드 텍스트·효과가 시트에 포함되는지.
