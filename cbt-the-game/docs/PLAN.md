## CBT 덱빌딩 게임 — 개발 계획 (PLAN, live)

> 이 노트가 엔트리 포인트. 돌아오면 이것부터 읽는다. 현재 상태 = "지금 어디인가".
> 기준 문서: [[CANON]] · [[SPEC-mechanics]] · [[SPEC-data]] · [[SPEC-screens]] · [[TASKS]]

### 목적

놀공이 게임 과몰입 치료용 덱빌딩 CBT 카드게임을 페이퍼 프로토타입으로 16차까지 빌드했고(2024.08~2025.03), 성대·놀공 플레이테스트에서 **실제 효과를 본 게임**이다. 이걸 디지털로 제대로 재구현한다. "충동 → 생각으로 억제 → 생각이 모여 실천 → 충동·과몰입 제거"를 직접 플레이로 겪게 하는, 규칙이 곧 CBT인 게임.

> **무시할 것**: 임상 빌드 `dev.mindshiftdtx.com`(Next.js, 삼성병원 임상 DTx)와 `250911 임상 기획서`. 별개 트랙, 캐넌 아님.

### 메인 incoming 폴더

`Assets/incoming/CBT Game/` — 캐넌 제작과정 기록(docx/pdf) + 카드 콘텐츠 시트 3종. 이 프로젝트 자료는 여기로.

### 방향 결정 (2026-06-22 확정)

1. **범위**: 1회기 덱빌딩 게임 파일럿 먼저. 코어 입증되면 6·7회기는 콘텐츠 교체로 재사용.
2. **기술**: 풀스택 웹앱이되 1단계는 manifestia식 store 추상화 + localStorage. 서버/DB는 2단계(화면 코드 불변, store 본문만 교체).
3. **메카닉 캐넌**: 페이퍼 프로토(도미니언식 자원 경제).
4. **승/패**: 9차 "칸 채우기"(실천 6칸 클리어 / 과몰입 6칸 실패). 16차 점수 시스템은 미채택.
5. **카드 콘텐츠**: 피터공 데이터 시트 3종에서 추출(placeholder 안 씀).

### 기술 스택

React 18 + react-router-dom 6 + Vite 5 (manifestia 골격 복제). store 추상화 레이어. B&W 미니멀 카드 UI. → 상세 [[SPEC-mechanics]].
