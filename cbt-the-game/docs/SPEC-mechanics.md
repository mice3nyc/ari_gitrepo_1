## SPEC — 게임 메카닉 / 상태 머신 (1단계)

> 기준: [[CANON]]. 태그 — `[캐넌]` 문서에 확정된 것 / `[1단계 결정]` 아리공이 코드 위해 정한 것(피터공 검토 대상) / `[2단계]` 미룸.
> 화면은 `store`만 호출한다. 승패 판정·상태 전이는 모두 store 안에서(화면 아님).

### 0. 보드 구조 `[캐넌]`

6 유형(K-IGD: salience/mood/tolerance/withdrawal/conflict/relapse) × 3 영역. 각 영역 6칸.
- **과몰입행동 칸** 6: 억제 실패가 쌓이는 곳. 6칸 다 차면 실패.
- **실천 칸** 6: 실천카드가 놓이는 곳. 6칸 다 차면 클리어.
- **생각 칸** 6: 모은 생각카드 누적 표시(같은 유형 2장 → 실천 전환 트리거).

### 1. 시작 상태 `[캐넌]`

- 시작 덱 = 충동카드 7장(플레이어 선택) + 생각하는힘 카드 3장.
- `[1단계 결정]` 선택 충동 7장은 K-IGD 6유형에 분산(한 유형 중복 허용). 보드 6칸 = 6유형이므로 7번째는 같은 유형 2장이 되는 셈. → **검토 포인트: 7장 vs 6유형 정합**(피터공 확인). 기본은 7장 유지, 유형 중복 허용.

### 2. 하루(턴) 루프

순서: `드로우 → 충동확인 → 생각카드 구매(억제) → 하루 정산 → 실천 전환 → 승패 체크 → 다음날`

- **드로우** `[캐넌 — 수치 검토]`: 덱에서 1장씩, 플레이어가 언제든 멈출 수 있음. 덱이 비면 버린덱을 섞어 보충(도미니언식). ⚠️ 하루 장수 불일치: 캐넌 docx "최대 5장" vs 10_21 시트 "4장 시작·최대 7장(강화 6/8)". 1단계 기본값은 **4장 시작·최대 7장**(시트 우선) — 검토 포인트.
- **생각카드 구매(억제)** `[캐넌 골격 / 1단계 결정 수치]`: 손에 든 생각하는힘으로 생각카드 구매. 구매한 생각카드와 **같은 유형의 손패 충동**이 억제되어 버린덱으로 이동.
  - `[1단계 결정]` 생각하는힘 1장 = 1코인. 생각카드 가격 = 2코인. (캐넌은 "가격에 맞는 카드만 구매" "더 높은 생각카드=더 높은 돈"으로 레벨 경제를 암시하나 1단계는 단일 가격으로 단순화. 피터공 검토.)
  - 같은 유형 충동이 손패에 없으면 생각카드 구매는 가능하되 즉시 억제 효과는 없음(생각 칸에만 누적). `[1단계 결정]`
- **하루 정산** `[캐넌]`: 억제 못 한(손패에 남은) 충동 → 해당 유형 과몰입행동 칸에 1 적립. **이미 과몰입 칸에 있던 같은 유형 충동을 그날도 못 막으면 과몰입행동 발동**(아래 4).
- **실천 전환** `[캐넌]`: 생각 칸에 **같은 유형 생각카드 2장**이 모이면 그 유형 실천카드 등장 → 실천 칸에 놓이고, 해당 유형의 **충동·과몰입행동을 영구 제거**(보드·덱에서 제외).
  - `[1단계 결정]` 실천카드 등장 = 자동(2장 충족 즉시). 캐넌 9차 아이디어("2장 중 더 와닿는 실천 선택")는 `[2단계]`.

### 3. 카드 상태 전이 `[캐넌]`

충동: `기본(active) → (같은 유형 생각 매칭) 억제(suppressed, 버린덱) → (해당 유형 실천 등장) 제거(removed)`. 억제 실패 시 `active` 유지 + 과몰입 칸 적립.

### 4. 과몰입행동 효과 `[캐넌 — cards-session1.json overuseEffect]`

10_21 시트가 유형별 발동 효과를 일반/강화 2티어로 명시. 두 계열:
- **ⓐ 현저성·내성·갈등**: 충동카드 1장 버린덱 추가(강화 2장) / 카드 4장 뽑고 시작·최대 7장(강화 6/8) / 생각카드 칸 1칸 축소(강화 생각+실천 칸 축소).
- **ⓑ 기분변화·금단·재발**: 충동카드 1장 버린덱 추가(강화 2장) / 억제 못한 충동이 즉시 과몰입행동이 됨(강화 생각카드로 억제 불가) / 보유 생각카드 1장 버림(강화 폐기).

`[1단계 결정]` 효과가 풍부하나 1단계 구현 범위는 **검토 포인트**(아래). 최소 = 충동 버린덱 추가 + 생각카드 칸 축소(ⓐ)/생각카드 버림(ⓑ). 칸 축소/즉시과몰입 등 강도 큰 효과는 단계적.

### 5. 승/패 판정 (store `checkEnd()`) `[캐넌]`

- 실천 칸 6칸 모두 채워짐 → `phase = cleared`.
- 과몰입행동 칸 6칸 모두 채워짐 → `phase = failed`.
- 정산·실천전환 직후 매번 체크.

### 6. store 상태 형태 (prefix `cbt_s1_`)

```jsonc
{
  "phase": "intro|select|playing|cleared|failed",
  "deck": [cardId], "hand": [cardId], "discard": [cardId],
  "dayDraws": [cardId],          // 이번 날 뽑은 카드 (≤5)
  "thinkingPower": 0,            // 보유 코인(손패 생각하는힘 환산)
  "board": {                     // 6 유형별
    "salience": { "overuse": 0, "thoughtCount": 0, "actionPlaced": false, "impulses": {id:"active|suppressed|removed"} },
    "...": {}
  },
  "day": 1
}
```

### 7. store 전이 함수 (화면이 호출할 인터페이스)

2단계 서버 교체 시 본문만 바뀌고 시그니처 유지. `[1단계 결정]` 1단계는 동기, 2단계 대비 Promise 반환 선설계 권장.

```
getGameState() → state
startGame()
selectImpulses(ids[7])           → {ok, error?}
drawCard()                       → {card, drawsLeft}      // ≤5 가드
stopDraw()                       → {hand}
buyThought(thoughtCardId)        → {result:'suppress'|'store-only'|'no-coin', affectedType}
resolveDay()                     → {overuseAdded[], triggered[]}
formActions()                    → {newActions[]}          // 2장 충족 유형 자동 실천
checkEnd()                       → {phase}
resetGame()
```

### 8. 검증 시나리오 (CDP) `[1단계 결정]`

- 클리어: 6유형 각각 같은 유형 생각 2장 구매·억제·실천 전환 → 실천 6칸 → cleared.
- 실패: 억제 방치 반복 → 과몰입 6칸 → failed.
- 뽑기 멈춤: 5장 전 stopDraw 동작, 6장째 드로우 차단.
- 영속: 새로고침 후 phase·board 복원.
- 런타임 예외 0.

### 결정 (2026-06-22, 아리공 — 피터공 "일단 결정해서 진행")

1. **충동 선택 = 6장(유형당 1장)**. 보드 6유형 = 실천 6칸 승리와 정합. 캐넌 7장 → 1단계 단순화. "각 유형에서 가장 와닿는 충동 하나씩."
2. **코인 = 생각하는힘 1코인 / 생각카드 1코인**. 억제는 기본 동작이라 접근 쉽게. 긴장은 "같은 유형 생각 2장 → 실천".
3. **드로우 = 4장 시작, 최대 7장, 멈춤 가능**(10_21 시트 우선).
4. **과몰입 효과 = 최소**: 미억제 충동이 과몰입 트랙 +1, 발동 시 충동카드 1장 덱 추가. 칸 축소·즉시과몰입·생각카드 폐기는 단계적.
5. **승/패 트랙**: 실천 6칸(유형별 실천 형성 시 +1) 채우면 클리어 / 과몰입 6칸(미억제 충동 누적) 채우면 실패. 두 트랙 레이스.
6. 유형 1:1 매칭 + 유형 미공개(내용 읽게) 유지.

> 모두 1단계 밸런스 가정. CDP 플레이로 stall/속도 확인 후 튜닝. 캐넌 충실(7장·레벨 경제 등) 복원은 피터공 검토 후.
