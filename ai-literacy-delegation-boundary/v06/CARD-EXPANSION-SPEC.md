> selfintro pilot 후 4 시나리오(groupwork·eorinwangja·career·studyplan)에 정점 + 카드 풀 박는 작업 spec. 백도 4개 병렬 launch용. 5/3 세션281.

## 작업 범위

대상 시나리오 yaml: `data/scenarios.yaml`

**작업 대상**: groupwork / eorinwangja / career / studyplan 중 1개 (백도별 1개 담당)

**Phase 1**: 묶음 안 정점(++/++) leaf 1개씩 박기
**Phase 2**: `domainPool` 신설 (특화 3 + 공통 4 = 7종)
**Phase 3**: `competencyCards` 27자리 박기

빌드·검증은 메인이 한다. 백도는 yaml 블록 텍스트만 출력.

## 캐논 (작업 전 숙지)

### 위임의 경계

위임은 도구가 아니라 **상태에서 그어진다**. AI 사용 자체가 부정적이 아니라, **할 수 있는 일을 위임하느냐 못하는 일을 위임하느냐**가 학습의 차이를 만든다. 자기 출발 후 적절한 AI 결합은 정점에 닿을 수 있다.

### 정점 - 회피 분포 (각 묶음)

각 tier1 묶음(A·B·C)에 다음이 있어야 한다:
- **정점 1개** — ++/++ (자기 주도 + 적절한 결합)
- **적절 1~2개** — +/+ 또는 0/+
- **부정·회피 1개** — -/- 또는 --/-

selfintro 모범:
- A 묶음: A1 정점(++/++) / A2 적절(+/+) / A3 얕음(0/+)
- B 묶음: B1 회피(--/-) / B2 중립(0/0) / **B3 정점(++/++)** ← 자기 주도+AI 검수
- C 묶음: **C1 정점(++/++)** ← 자기 자료 적극 / C2 부정(-/-) / C3 회피(--/-)

## Phase 1 — 묶음 안 정점 점검

각 묶음에 ++/++ leaf 있는지 점검. 없으면 묶음 안 가장 어울리는 leaf를 정점으로 격상.

### 정점 격상 시 정정 자리 (4곳)

#### 1. tier2 leaf 자체 (delegation·knowledge → ++/++)

```yaml
- id: B3   # 예시
  label: [정점 메시지에 맞춰 텍스트 강화]
  delegation: '++'
  knowledge: '++'
  lesson: [정점 캐논 메시지]
```

**중요**: 4 시나리오는 단일 점수 구조 유지(매트릭스 X). selfintro만 매트릭스(`delta.afterA/afterB/afterC`).

#### 2. results.{leaf} basePoint 격상

```yaml
B3:
  text: [정점 결과 강조 — 자기 주도 + 검수 깊음]
  basePoint: 88     # 격상 (이전 60~70대)
  deltaR2: 5
  deltaR3: 12
  summary: [정점 한 줄]
  lesson: [정점 캐논]
```

#### 3. reviewSupplements (R3 점수 표기 정합)

```yaml
{leaf}R3: ... 점수 +12.   # deltaR3 12와 정합
```

#### 4. finals.{leaf}R1·R2·R3 (3개 자리)

```yaml
B3R1: { score: 88, grade: S, item: ..., delegation: ++, knowledge: ++, awareness: [정점 메시지] }
B3R2: { score: 93, grade: S, item: ..., delegation: ++, knowledge: ++, awareness: [검수 정점 메시지] }
B3R3: { score: 100, grade: S, item: ..., delegation: ++, knowledge: ++, awareness: [완성 정점 메시지] }
```

격상 점수 식: basePoint 88 + R1=0, R2=+5, R3=+12 → 88, 93, 100.

## Phase 2 — domainPool 신설

### 공통 풀 (모든 시나리오 공유) — 4종

- **자기검증** — 자기 결과물 다시 보는 능력
- **자기성찰** — 자기 행동·결정 돌아보는 능력
- **검수능력** — 결함 발견하는 능력
- **비판적 사고** — 정보·결과 의심하고 따져보는 능력

### 시나리오 특화 풀 (담당 시나리오별)

- **groupwork**: 협업능력 / 소통능력 / 발표력
- **eorinwangja**: 문해력 / 공감력 / 해석능력
- **career**: 진로탐색 / 정보분석 / 직업이해
- **studyplan**: 학습설계 / 시간관리 / 학습전략

### domainPool 박는 자리

각 시나리오 yaml 상단 `id` 다음 줄에:

```yaml
groupwork:
  id: groupwork
  ...
  domainPool: [협업능력, 소통능력, 발표력, 검수능력, 자기검증, 자기성찰, 비판적 사고]
```

순서: 시나리오 특화 3 → 공통 4.

## Phase 3 — competencyCards 27자리 박기

### 자리 룰

| 자리 | R1 | R2 | R3 |
|---|---|---|---|
| **회피** (B1·C3·기타 통째 위임) | 0장 | 0장 | 0장 |
| **부정** (예: C2 baseline) | 0장 | 1장 (비판적 사고) | 3~4장 (회복) |
| **얕음** (예: A3) | 1장 (특화) | 2장 | 3~4장 (자기성찰) |
| **중립** (예: B2) | 1장 (특화) | 2~3장 (검수) | 3~4장 (자기검증) |
| **적절** (+/+ leaf) | 2장 (특화 둘) | 3장 (+검수) | 4장 (+자기검증) |
| **정점** (++/++ leaf) | **3장** (특화 둘 + 공통 1) | **4장** (+비판적사고) | **5장** (+자기검증·자기성찰) |

### 핵심 룰

- **회피 자리는 0장** — 학습 거의 없음의 정합
- **노력 자리는 R1부터 1장 이상 보장** — 자기 손으로 했으면 카드 받아야 (피터공 5/3 진단)
- **R3는 풍부** — 전면 검수 = 자기검증·자기성찰 자리
- **정점은 R1부터 풍부** — 자기 주도 + 검수 = 카드 폭발

### selfintro 모범 패턴 (참조)

```yaml
selfintro:
  competencyCards:
    A1R1: [자기지식, 정리력]                                    # 정점 R1
    A1R2: [자기지식, 정리력, 검수능력]                          # 정점 R2 (+검수)
    A1R3: [자기지식, 정리력, 표현력, 자기검증]                  # 정점 R3 (+자기검증)
    A2R1: [자기지식, 표현력]                                    # 적절 R1
    A2R2: [자기지식, 표현력, 검수능력]                          # 적절 R2
    A2R3: [자기지식, 표현력, 검수능력, 자기검증]                # 적절 R3
    A3R1: [자기지식]                                            # 얕음 R1 — 1장 보장
    A3R2: [자기지식, 정리력]                                    # 얕음 R2
    A3R3: [자기지식, 정리력, 자기성찰, 검수능력]                # 얕음 R3 (+자기성찰)
    B1R1: []                                                    # 회피 — 0장
    B1R2: []
    B1R3: []
    B2R1: [자기지식]                                            # 중립 R1
    B2R2: [자기지식, 표현력, 검수능력]                          # 중립 R2
    B2R3: [자기지식, 표현력, 검수능력, 자기검증]                # 중립 R3
    B3R1: [자기지식, 표현력, 검수능력]                          # 정점 R1 — 3장
    B3R2: [자기지식, 표현력, 검수능력, 비판적 사고]             # 정점 R2 — 4장
    B3R3: [자기지식, 표현력, 정리력, 검수능력, 자기검증]        # 정점 R3 — 5장
    C1R1: [자기지식, 정리력, 자기성찰]                          # 정점 R1 — 3장
    C1R2: [자기지식, 정리력, 자기성찰, 비판적 사고]             # 정점 R2 — 4장
    C1R3: [자기지식, 정리력, 표현력, 자기성찰, 자기검증]        # 정점 R3 — 5장
    C2R1: []                                                    # 부정 R1 — 0장
    C2R2: [비판적 사고]                                         # 부정 R2 — 회복 시작
    C2R3: [자기지식, 정리력, 자기성찰, 자기검증]                # 부정 R3 — 회복
    C3R1: []                                                    # 회피 — 0장
    C3R2: []
    C3R3: []
```

## 출력 형식 (백도 → 메인)

각 백도는 yaml 블록 텍스트로 출력. 메인이 회수해서 직접 yaml에 박음.

### 출력 1 — Phase 1 (정점 격상)

격상한 leaf의 자리별 yaml 블록:

```
### tier2 leaf
- id: B3
  label: ...
  delegation: ++
  knowledge: ++
  lesson: ...

### results.B3
B3:
  text: ...
  basePoint: 88
  deltaR2: 5
  deltaR3: 12
  summary: ...
  lesson: ...

### reviewSupplements
B3R3: ... 점수 +12.

### finals.B3R1·R2·R3
B3R1: { ... }
B3R2: { ... }
B3R3: { ... }
```

격상 leaf가 묶음별로 0~3개. 안 격상한 묶음은 "기존 정점 leaf 명시" + "정정 없음" 표시.

### 출력 2 — Phase 2 (domainPool)

```yaml
domainPool: [특화1, 특화2, 특화3, 검수능력, 자기검증, 자기성찰, 비판적 사고]
```

### 출력 3 — Phase 3 (competencyCards)

27자리 yaml 블록 그대로:

```yaml
competencyCards:
  A1R1: [...]
  A1R2: [...]
  ...
  C3R3: []
```

### 백도 보고 (간결)

- 격상한 정점 leaf 묶음별 (예: B 묶음 B2 → 정점 / C 묶음 C1 그대로)
- 새 카드 자리 충전한 자리 수
- 의심 자리 (애매한 leaf 1~2개) 메인 검토 요청
