#### 1. 시나리오 검수 결정 3건 (4/28 세션251 검수 후 피터공 결정)

- 발견: A1R3 = 100점 (시나리오4·5). 카드/레벨업 보너스 누적 시 100 초과 가능.
- 피터공: "상한 두지 말고 계속 해보자."
- 적용: score 상한 처리 없음. 누적 그대로.

**1.2 시나리오4 C2R3 = A 등급(70점) 유지**

- 피터공: "플테에서 여러 상황을 검증해 볼 수 있는 것에 무엇이 좋을지를 기준으로 아리공이 판단."

**1.3 timeBudgetSec 데드 필드 — v0.6 정돈 사이클로 미룸**

#### 2. 시나리오 선택 UI — 옵션 C 하이브리드 (4/28 세션251)

**2.1 시나리오 노출 정책**
- 후보 A: 시작 화면에 5장 카드 (자유 선택)

- 적용: 5 시나리오 카드 + "다음 추천 →" 강조 (`.scenario-card.next`) + 다른 시나리오도 클릭 가능.

**2.2 시나리오 1회만 시도 가능**
- 피터공: "시나리오는 일단은 일회만 시도 가능하도록."
- 적용: `gameState.clearedScenarios` 배열 신설. 클리어한 시나리오는 카드 disabled (line-through + cursor:default).
- 영향: PLAN의 "다시 도전하기"(누적 유지) 버튼은 의미 변경 → "시나리오 선택으로"로 대체. `scenarioRepeatCount`는 사실상 0~1만 가능, 누적 메커닉(D-07)은 v0.6 이후로 미룸.

**2.3 학기 진행 프로그레스 UI**
- 피터공: "진행 프로그레스가 잘 인식되도록 UI를 만들어야겠는데."
- 적용:

#### 5. 레벨업 메커닉 — 단계별 + exp 리셋 (4/28 세션251)

- 부수효과: 잉여 exp 손실 (예: Lv1+45 → Lv2 진입, 25 exp 손실)

**5.2 expThresholds 조정 [0,20,50,100,200] → [0,20,40,65,100]**

  - 변경 전: S5 후 Lv4 머무름 (Lv5 도달 X)
  - 변경 후: S4에서 Lv5 도달 ✓ (S5는 머무름)

- 위임/우회 위주(B1, C 분기): Lv2~3

#### 7. 회복 없는 학기 + 카드 시스템 진입 (4/29 세션254 — 피터공 디자인 사유 후 결정)

**진입 배경**: 4/29 단계별 자원 소비 + cost meter 가시화 직후 피터공 4축 사유 제기. "비용이 메시지가 되니 선택이 흥미로워졌다. 가능성이 있어." → 자원 회복 모델 + 레벨 점수 영향 메커니즘 재설계.

요청 노트 원본: [[요청.26.0429.1925-v05회복없는학기]]

##### 7.1 자원 회복 모델 — 회복 없음 (학기 메타포)

- **피터공 진단**: "쓴만큼 채워지면 무의미하지." (현 SPEC §2.3: 70% + 등급 보너스 10~60% → 등급 ↑일수록 회복 ↑ → 무한 자원 사이클).
- **고려한 모델**:
  - (a) 회복 없음 — 학기 = 한정된 풀
  - (b) 부분 고정 회복 (등급 무관)
  - (c) 시간 ≠ 에너지 차별화 (시간만 회복)
  - (d) 카드 효과로 회복
- **결정**: **(a) 회복 없음** + 학기 메타포 + (d) 카드로 효율 ↑.
- **근거**: 무한 사이클 차단 + AI 리터러시 주제 직결. "학기는 한정되어 있다. 다만 같은 시간을 더 잘 쓰는 법을 배운다."
- **코드**: `CONFIG.autoRecoverOnEnd=false`. `goCut6`에서 `recoverResources` + `applyLevelUpMeterIncrease` 호출을 가드로 감쌈. SPEC §2.3은 v0.5 한정으로 OFF (v0.6 이후 재검토 가능).

##### 7.2 비용 multiplier — 0.6 (시작값, 플테 후 조정)

- **피터공 요청**: "적정 선택 시 1.5~2.5 시나리오 버텨야."
- **현 데이터(어린왕자)**: 시간 평균 49 (max 100), max=100 한 번 잘못 선택 즉사 위험.
- **결정**: `CONFIG.resourceCostMultiplier=0.6`. 데이터 135 entries는 보존, 단계 헬퍼 출력에서 한 군데 곱셈.
- **결과**: 어린왕자 시간 평균 29 / 에너지 23, max 60. 100 자원으로 평균 ~3.4 시나리오 (신중 선택 시), 빡빡한 선택은 1~2 시나리오.
- **튜닝 정책**: multiplier 한 줄 = 언제든 0.7/0.5 등으로 즉시 조정. 데이터는 stable하게 유지.

##### 7.3 레벨 → 점수 모델 — 카드 시스템 (검토 격상 폐기)

- **이전 (SPEC §3.4 C안)**: 레벨이 검토 인덱스를 격상. R1 선택해도 Lv3+면 R2 효과 + Lv4·5 추가 점수.
- **피터공 새 가설**: "경험+ = 역량+로 나중에 AI를 효과적으로 사용해서 시간/에너지 적게 쓰고 점수는 높일 수 있다? 그렇다면 내가 그런 역량 상태라는 것을 알 수 있어야 한다. 경험치마다 획득하는 아이템 카드로 위임 효과성을 높이거나 짧은 시간에 더 많은 검토를 할 수 있게 되는 것을 구체적으로 표현해 주는 건 어떨까?"
- **결정**: 카드 시스템 도입. 검토 격상 메커닉(C안) 폐기 — 추상적이고 체감 어려움. 카드 = 구체적 효과 명시 = "내 역량 상태 가시화."
- **카드 예시 (Lv 마일스톤)**:
  - Lv2 — "프롬프트 정교화": B 분기(AI 위임) 시간 비용 -10%
  - Lv3 — "교차 검증 습관": R3 검토 비용 -3, 결과 점수 +3
  - Lv4 — "맥락 파악력": A 분기(직접) 결과 점수 +5
  - Lv5 — "AI 능숙": 모든 비용 -15%, 모든 결과 점수 +10
- **근거**: 게임 루프 단순화 — 시간/에너지 비용 → 결과 등급 → 경험치 → 레벨업 → 카드 획득 → 다음 시나리오부터 비용/점수 효과 → "어떤 학습자가 됐는가." AI 리터러시 주제와 직결.
- **코드**: `CONFIG.useReviewLevelBoost=false` (즉시) + `CARD-SPEC.md` 신설(작업 대기) + 카드 효과 hook (consumeStage / calculateFinalScore) — 다음 회기.

##### 7.4 레벨업 곡선 — 현 [0,20,40,65,100] 유지

- **피터공 우려**: "레벨업이 너무 빠른 느낌."
- **진단**: 자연 플레이 = 학기 5번에 평균 12 exp/회 → 누적 60 = Lv3 도달이 의도. 디버그 max=25 누름이 빨라 보이는 원인. (DECISIONS §5.2 시뮬 — S5 후 Lv5 도달, A 평균 시 Lv4, B/C 위주 시 Lv2~3.)
- **결정**: 곡선 유지. 카드 시스템 도입 후 재평가.

##### 7.5 학맞통 카드(v0.3 계승) ↔ 역량 카드(신규) — 별도 트랙

- **결정**: 두 시스템 분리 운영.
  - 학맞통 카드 = 시나리오별 결합 슬롯 (cardSlots) — leaf 결과 boost (cardBoost: 20). v0.3 보존.
  - 역량 카드 = 학기 누적 마일스톤 — 다음 시나리오부터 비용/점수 효과 적용. v0.5 신규.
- **근거**: 두 시스템이 다른 시간 스케일 + 다른 효과 영역. 통합 시 복잡도 증가 + 메커닉 충돌.

##### 7.6 결정 처리 정책
