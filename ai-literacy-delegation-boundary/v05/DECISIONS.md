#### 7. 회복 없는 학기 + 카드 시스템 진입 (4/29 세션254 — 피터공 디자인 사유 후 결정)

**진입 배경**: 4/29 단계별 자원 소비 + cost meter 가시화 직후 피터공 4축 사유 제기. "비용이 메시지가 되니 선택이 흥미로워졌다. 가능성이 있어." → 자원 회복 모델 + 레벨 점수 영향 메커니즘 재설계.

요청 노트 원본: [[요청.26.0429.1925-v05회복없는학기]]

##### 7.1 자원 회복 모델 — 회복 없음 (학기 메타포)

- **피터공 진단**: "쓴만큼 채워지면 무의미하지." (현 SPEC §2.3: 70% + 등급 보너스 10~60% → 등급 ↑일수록 회복 ↑ → 무한 자원 사이클).
- **고려한 모델**:
  - (a) 회복 없음 — 학기 = 한정된 풀
  - (b) 부분 고정 회복 (등급 무관)
  - (c) 시간 ≠ 에너지 차별화 (시간만 회복)
  - (d) 카드 효과로 회복
- **결정**: **(a) 회복 없음** + 학기 메타포 + (d) 카드로 효율 ↑.
- **근거**: 무한 사이클 차단 + AI 리터러시 주제 직결. "학기는 한정되어 있다. 다만 같은 시간을 더 잘 쓰는 법을 배운다."
- **코드**: `CONFIG.autoRecoverOnEnd=false`. `goCut6`에서 `recoverResources` + `applyLevelUpMeterIncrease` 호출을 가드로 감쌈. SPEC §2.3은 v0.5 한정으로 OFF (v0.6 이후 재검토 가능).

##### 7.2 비용 multiplier — 0.6 (시작값, 플테 후 조정)

- **피터공 요청**: "적정 선택 시 1.5~2.5 시나리오 버텨야."
- **현 데이터(어린왕자)**: 시간 평균 49 (max 100), max=100 한 번 잘못 선택 즉사 위험.
- **결정**: `CONFIG.resourceCostMultiplier=0.6`. 데이터 135 entries는 보존, 단계 헬퍼 출력에서 한 군데 곱셈.
- **결과**: 어린왕자 시간 평균 29 / 에너지 23, max 60. 100 자원으로 평균 ~3.4 시나리오 (신중 선택 시), 빡빡한 선택은 1~2 시나리오.
- **튜닝 정책**: multiplier 한 줄 = 언제든 0.7/0.5 등으로 즉시 조정. 데이터는 stable하게 유지.

##### 7.3 레벨 → 점수 모델 — 카드 시스템 (검토 격상 폐기)

- **이전 (SPEC §3.4 C안)**: 레벨이 검토 인덱스를 격상. R1 선택해도 Lv3+면 R2 효과 + Lv4·5 추가 점수.
- **피터공 새 가설**: "경험+ = 역량+로 나중에 AI를 효과적으로 사용해서 시간/에너지 적게 쓰고 점수는 높일 수 있다? 그렇다면 내가 그런 역량 상태라는 것을 알 수 있어야 한다. 경험치마다 획득하는 아이템 카드로 위임 효과성을 높이거나 짧은 시간에 더 많은 검토를 할 수 있게 되는 것을 구체적으로 표현해 주는 건 어떨까?"
- **결정**: 카드 시스템 도입. 검토 격상 메커닉(C안) 폐기 — 추상적이고 체감 어려움. 카드 = 구체적 효과 명시 = "내 역량 상태 가시화."
- **카드 예시 (Lv 마일스톤)**:
  - Lv2 — "프롬프트 정교화": B 분기(AI 위임) 시간 비용 -10%
  - Lv3 — "교차 검증 습관": R3 검토 비용 -3, 결과 점수 +3
  - Lv4 — "맥락 파악력": A 분기(직접) 결과 점수 +5
  - Lv5 — "AI 능숙": 모든 비용 -15%, 모든 결과 점수 +10
- **근거**: 게임 루프 단순화 — 시간/에너지 비용 → 결과 등급 → 경험치 → 레벨업 → 카드 획득 → 다음 시나리오부터 비용/점수 효과 → "어떤 학습자가 됐는가." AI 리터러시 주제와 직결.
- **코드**: `CONFIG.useReviewLevelBoost=false` (즉시) + `CARD-SPEC.md` 신설(작업 대기) + 카드 효과 hook (consumeStage / calculateFinalScore) — 다음 회기.

##### 7.4 레벨업 곡선 — 현 [0,20,40,65,100] 유지

- **피터공 우려**: "레벨업이 너무 빠른 느낌."
- **진단**: 자연 플레이 = 학기 5번에 평균 12 exp/회 → 누적 60 = Lv3 도달이 의도. 디버그 max=25 누름이 빨라 보이는 원인. (DECISIONS §5.2 시뮬 — S5 후 Lv5 도달, A 평균 시 Lv4, B/C 위주 시 Lv2~3.)
- **결정**: 곡선 유지. 카드 시스템 도입 후 재평가.

##### 7.5 학맞통 카드(v0.3 계승) ↔ 역량 카드(신규) — 별도 트랙

- **결정**: 두 시스템 분리 운영.
  - 학맞통 카드 = 시나리오별 결합 슬롯 (cardSlots) — leaf 결과 boost (cardBoost: 20). v0.3 보존.
  - 역량 카드 = 학기 누적 마일스톤 — 다음 시나리오부터 비용/점수 효과 적용. v0.5 신규.
- **근거**: 두 시스템이 다른 시간 스케일 + 다른 효과 영역. 통합 시 복잡도 증가 + 메커닉 충돌.

#### 9. 자원 모델 2차 조정 — 부분 회복 + 0 floor + cost 다양성 (4/29 22:14 피터공 발화, 4/30 작업 예정)

**진입 배경**: §7.1 "회복 없음" 결정 후, 피터공 재검토. "회복 없음"이 학기 메타포로는 작동하지만, 성취 보상 루프가 끊겨 메시지가 한쪽으로 치우친다. 성장의 가시화(레벨업·점수)가 자원으로도 돌아와야 "같은 시간을 더 잘 쓰는 법"이 학습자에게 체감된다.

##### 9.1 부분 자원 회복 — 레벨업 + 점수 기반 ⛔ 폐기 (4/30 세션256, §10.1 참조 — 8.7 코드 rollback 완료 세션257)

- **피터공**: "레벨이 올라가면 시간과 에너지 자원도 일부 복구 되어야 한다. 결과물 점수에 따라서도 일부 복구 되어야 함."
- **방향**: §7.1 "회복 없음(autoRecoverOnEnd=false)" 부분 후퇴. 회복 트리거를 두 축으로 분리.
  - (A) 레벨업 회복: didLevelUp 분기에서 일정량 회복 (구체 수치는 발란스)
  - (B) 점수 회복: 사이클 종료 시 등급에 따라 회복 (S/A/B/C/D 차등)
- **잠정 SPEC 후보** (플테 후 조정):
  - 레벨업: time/energy 각 +15 (현 multiplier 0.6 기준 leaf 평균비용 ~25의 60%)
  - 점수 회복: S=+20 / A=+12 / B=+6 / C=0 / D=0 (또는 -2 페널티)
  - 최대치는 100 (max 변동 없음 — meterMaxByLevel 별도)
- **§7.1과의 관계**: `autoRecoverOnEnd=false` 유지하되, 새로운 회복 hook 두 개 추가. 즉 "쓴만큼 일률 회복"은 여전히 OFF. "성취 회복"만 ON.
- **학기 메타포 보존**: 회복량 < 평균 소비량으로 설정 → 무한 사이클 차단은 유지.

##### 9.2 자원 0 floor — clamp 필수

- **피터공**: "자원은 0 이하로 내려가면 안됨."
- **현재 동작 점검 필요**: consumeStage가 음수로 내려가는지 코드 확인.
- **방향**: `Math.max(0, current - cost)` clamp 적용. 단, 추가 결정 필요 항목:
  - (a) 자원 0이면 해당 사이클 어떻게? (강제 종료 / D 등급 처리 / 다음 사이클로 진행 가능?)
  - (b) cost > 잔여자원이면 선택지 비활성화? (UI 가시화)
  - (c) 0 floor와 §9.1 점수 회복의 상호작용 (D 등급 = 회복 0 = 다음 사이클도 0 시작 → 데드락 방지 장치)
- **4/30 결정 대기**: 위 (a)(b)(c)를 피터공과 합의 후 구현.

##### 9.3 선택지 cost 다양성 — 다축 trade-off

- **피터공**: "발란스 조정이 필요하겠지만, 선택지 중에는 어떤 것은 시간이 더 들고 에너지는 적게 드는 것, 에너지 적게 들고 시간 오래 걸리는 것 같은 다양성이 있어야 함."
- **진단 필요**: 현 어린왕자 시나리오 27 leaf의 시간/에너지 cost 분포 확인. 균질하면 "선택의 맛" 약화.
- **방향**: leaf별로 (시간↑, 에너지↓) / (시간↓, 에너지↑) / (양쪽 ↑↑) / (양쪽 ↓↓) 4 축에 분포되도록 데이터 재조정.
  - 의미: A 분기(직접) = 보통 시간↑ 에너지↑ / B 분기(AI) = 시간↓ 에너지↓ but 결과 분산 / C 분기(타인) = 시간↑ 에너지↓ 같은 패턴 가능
  - cost meter UI는 이미 두 축 분리 표시 — 데이터만 다양화하면 즉시 가시화
- **§7.2 multiplier와 별개**: multiplier는 전체 스케일 조정, §9.3은 leaf 간 상대값 조정.
- **작업**: SCENARIOS의 resourceCosts JSON 수정 (시나리오 5개 × 27 leaf = 135 entries). 백도 위임 가능.

1. **진단**: 현 27 leaf cost 분포 분석 (균질도 측정) + consumeStage clamp 동작 확인
2. **9.2 결정**: 자원 0 처리 정책 (a/b/c) 피터공과 합의
3. **9.1 구현**: 레벨업 회복 + 점수 회복 hook 추가
4. **9.2 구현**: clamp + UI 비활성화 + 데드락 방지

##### 10.4 자원토큰 사용처 — 시간/에너지 2축 전용 (4/30 세션257 결정)

| 항목 | 토큰 비용 | 효과 |

**2축 단순화 결정 (4/30 세션257)**: 초기 4축안(위/도 영구 투자 옵션 +1=3토큰)에서 2축으로 축소. 피터공 사유 — "30 토큰 받으면 시간 20+에너지 10에 충전"하는 직관적·만족스러운 UX. 위/도 변동은 별도 채널(§10.6)로 분리 = 결정 노드는 분배 팝업 한 군데에 집중. 4축안의 "장기 역량 투자" 결정 노드는 폐기.

**잠정 cap**: 한 항목에 0~보유토큰 자유. 메터 max 초과 충전은 자동으로 max에서 잘림(잉여는 손실). UI에서 미리 max 표시 + 초과분 회색.

##### 10.5 자원토큰 분배 팝업 — 신규 UI (8.10a)

- 컷 6 결과 + (레벨업 시 §10.11 팝업) 직후 강제 진입 (모달)
- 시간/에너지 2축 [-]/[+] 컨트롤 + 남은 토큰 카운터
- **남은 토큰이 0이 되어야 [다음 시나리오 →] 활성화** — use it or lose it
- 분배 미리보기: "→ 시간 +20 / 에너지 +10 충전" (피터공 예시)
- 시각: 토큰을 두 양동이(시간/에너지)에 나눠 담는 비주얼. 만족스럽고 직관적. 메터가 차오르는 애니메이션
- 메터 max 초과 충전은 자동 잘림 — 미리 회색 표시로 손실 방지 유도
- **+/- 버튼 hold-to-repeat (4/30 세션257 1차 플테 후 추가)**: 누르고 있으면 350ms 후 80ms 간격으로 연속 증감. 자원 30+ 토큰 받아도 사용 편의. `bindHoldRepeat` 헬퍼
- 발란스 시뮬(2축): 평균 B + 레벨업 2회 학기 = 5×6 + 2×20 = 70 토큰. 누적 자원 소비 ~300 대비 23%. 4축 시뮬보다 자원 회복 비중 ↑ → 발란스 8.12에서 적립값 재평가 필요

##### 10.6 위/도 자동 할인 메카닉 + 변동 채널 (4/30 세션257 명확화)

- **위임 선택력 N점** = 모든 선택지 시간 비용 **-N×배율** 자동 차감
- **도메인 지식 N점** = 모든 선택지 에너지 비용 **-N×배율** 자동 차감
- **효과 배율 (4/30 세션257 8.12 발란스 1차)**: `CONFIG.competencyDiscountMult` 도입. **잠정 3**
  - 사유: 첫 플테에서 위/도 1~3 누적 + 비용 30 → 효과 -1~-3 = 미미. "empowering 느낌 없음" (피터공)
  - 배율 3 적용: 위 +3 → 시간 -9 (raw 30 → 21, 30% 할인 = 체감)
  - 음수 -3 페널티: 시간 +9 (30 → 39) — 강한 압박, "역량 음수면 게임 어려워짐" 메카닉 강화
  - 학기 끝까지 ++만 → +10 → -30 (비용 0 가까이) / 학기 끝까지 --만 → -10 → +30 (난이도 급등)
  - 박스 표시는 **배율 적용된 효과량** ("위 +3 → 박스 -9", 머리로 raw - 9 = 최종 비용)
  - 추가 발란스: 8.12 플테 후 미세 조정 (2~4 사이)
- 매핑 의미: "AI 잘 다루는 능력 → 시간 절약 / 도메인 지식 → 에너지 절약" (AI 리터러시 메타포 직결)
- hook 위치: `getTier1Cost / getTier2Cost / getReviewCost` 3개 함수에서 multiplier 적용 후 위/도 차감

**비용 표시 디자인 — 4박스 그리드 (4/30 세션257 피터공 디자인 결정)**:

```
[시간 비용]  [위임판단력]   |   [에너지 비용]  [도메인지식]
   ┌──┐        ┌──┐        |       ┌──┐         ┌──┐
   │ 3│        │-2│        |       │ 5│         │-1│
   └──┘        └──┘        |       └──┘         └──┘
   (빨강)      (초록)              (빨강)        (초록)
```

- **시간/에너지 비용 박스**: 큰 빨강 숫자 = **raw cost** (multiplier 적용 후, 위/도 차감 **전**)
- **위임판단력/도메인지식 박스**: 효과 표시
  - 양수 (할인): 초록 `-N` (비용에서 N 깎이는 효과)
  - 음수 (페널티): 빨강 `+|N|` (비용에 N 추가되는 효과)
  - 0: 회색 점선 박스, 흐린 `0` (자리 유지)
- **세로 구분선**: 시간 그룹 / 에너지 그룹 분리
- **기존 progress bar 제거** — 숫자만 강조 (피터공 의도)
- **최종 비용은 시스템이 알아서 계산** (consumeStage가 차감 후 cost로 처리). 플레이어는 raw + 효과를 보고 머리로 합산 — 역량이 비용에 어떻게 작용하는지 직접 체감
- 라벨 표기: "위임판단력" / "도메인지식" (피터공 비용 라벨용 신표현). 결과 화면 등 기존 "위임 선택력" 표현과의 통일은 후속 결정

**위/도 변동 채널 (4/30 세션257)**:
- 위/도는 **시나리오 선택 결과로만 자동 변동** — 선택지 결과의 delegationChoice/knowledge delta 누적, B1 던지기 누적 → 음수
- **자원토큰으로 살 수 없음** (§10.4 2축 결정에 따라 4축안 폐기)
- 레벨업 시 자동 +X 부여 여부는 **§10.11에 보류** — 플테 후 결정. 현재는 영향 없음

##### 10.7 음수 역량 — 허용 + 대칭적 비용 증가

- 피터공 사유: "물론 위임을 연속하면 역량이 줄어드는 것이 맞지. 음수 있어야 할 것 같아. 음수 경우에는 결과에 전반적으로 악영향을 주는게 상식적이고 직관적"
- **역량은 영구**(한 학기 안). 임시 X
- **음수 허용** — B1 같은 위임 던지기 누적하면 역량 음수
- **대칭 효과**:
  - 위 +3 → 시간 비용 -3 (할인)
  - 위 -3 → 시간 비용 **+3** (페널티, 비용 증가)
- "결과에 악영향" = 자원 빨리 소진 → 후속 시나리오 선택지 제한 → 결과 악화. 메카닉이 자연스럽게 흐름

##### 10.13 학기 종합 리포트 재설계 — 카툰 시리얼 (4/30 세션257 1차 플테 후)

**현재 약점 (피터공 진단)**:
- 통계 박스 6개 — **평균 점수는 무의미**. 카드/아이템은 미구현(카드)+의미 없음(아이템)
- 시나리오별 결과 = 텍스트 한 줄씩 — "다섯 결정의 발자국"이 약함
- 위/도 누적의 의미가 안 박힘 — 단순 +5 / -2 숫자만, 해석 없음

**재설계 디자인**:

1. **상단 통계 — 4박스로 축소**
   - 학기 총점 / 최종 레벨 / 위임 선택력 / 도메인 지식
   - ~~평균 점수~~ 제거
   - ~~카드 / 아이템~~ 제거 (Phase 7 카드 시스템 미구현, v0.6에서 부활)

2. **상단 추가 — 위/도 의미 분석 박스**
   - 누적 위 + 누적 도 값에 따른 해석 텍스트
   - 잠정 4유형 분류 (양수/음수 조합):
     - 위 +, 도 + → "AI를 동료로 쓰면서 자기 도메인도 키운 학생"
     - 위 +, 도 − → "AI에 의존했지만 자기 학습은 부족한 학생"
     - 위 −, 도 + → "자력으로 풀어낸 학생, AI 활용은 적었음"
     - 위 −, 도 − → "AI도 자기도 활용하지 않은 학기. 다시 한 번"
   - 0 근처(±2 이내)는 "흔들리며 길을 찾는 중"
   - 텍스트 정확도는 8.13 플테로 다듬기

3. **하단 — 카툰 시리얼 (NEW, 기존 텍스트 결과 대체)**
   - 5 시나리오를 좌우 흐름 카툰 패널로
   - 각 시나리오 1~2장의 컷 이미지 + 무드 대사/설명
   - 마지막 프레임으로 시상 이미지 (`scenario_result_{X}.png`)
   - "전체 결과가 카툰 시리얼로 나온 느낌" (피터공)

##### 10.14 학기 종합 리포트 코믹스트립 + 메시지 캡션 (4/30 세션260)

8.13 카툰 시리얼 위에 두 가지 결정 추가:

1. **결과 화면 노란 한 줄 LV UP 박스 제거** (`showFinalReport` line 2820 부근)
   - 같은 시점에 8.10b 레벨업 모달이 더 강하게 정보 전달 → 결과 패널 안 노란 박스는 시각 잡음
   - 코드: `if(didLevelUp){ ... }` 블록 삭제, 주석으로 흔적만 남김

2. **컷별 메시지 캡션 (코믹스트립 형식)** — 한 컷 = 이미지 + 캡션 panel
   - CSS 신설: `.comic-panel` (column flex, width:160px) + `.comic-caption` (font 12px, line-height 1.5, word-break:keep-all) + `.comic-caption b` (display:block, font-size 13px, 색·굵기 강조) + `.comic-caption-mood` (검정 윗선 + 굵음)
   - 시상 panel에 mood 통합 (이전: strip 끝에 별도 한 줄 → 변경: 시상 panel caption)
   - 컷별 텍스트 차원:
     - cut1: 시나리오 도입 (title 굵게 + situation 첫 문장)
     - cut2: 1차 선택 카테고리 (위임/직접/참고) — `tier1.lesson`
     - cut3: 그 선택의 이 상황에서의 의미 — `tier2.lesson`
     - cut4: 결과 + 레슨 — `results.summary`(굵게) + `results.lesson`
     - cut5: 검토 → 도메인지식 변화 — `reviews.lesson` (도메인지식 +N 결과 표시) + `reviewSupplements[leaf]` (회상 narration)
     - 시상: `CONFIG.resultMoods[grade]` (mood 대사)

3. **메시지 두 축**
   - **위임판단력**: AI에 시킬 일과 안 시킬 일 구분 (cut2~4)
   - **도메인지식**: 자기 깊이·지식·이해를 쌓는 것 (cut5에 +N 결과 표시)
   - 톤 패턴: "행동 — 결과/의미" 한 줄 (em dash 연결)
   - "점수" 단어 빼고 "도메인지식 +N" 결과 느낌 (피터공 4/30 결정)

4. **시나리오 데이터 확장 — `lesson` 필드 24개씩 5 시나리오 = 120개 추가**
   - tier1[i].lesson (3) + tier2[][].lesson (9) + results[].lesson (9) + reviews[i].lesson (3) = 24
   - 5 시나리오 동일 패턴 — selfintro 톤을 base로 백도 4 시나리오 채움 (4/30 세션260)
