#### 1. 시나리오 검수 결정 3건 (4/28 세션251 검수 후 피터공 결정)

ChatGPT 시나리오 1·2·4·5 검수 통과 후 발견 3개 ⚠️ 해소.

**1.1 점수 상한 없음**
- 발견: A1R3 = 100점 (시나리오4·5). 카드/레벨업 보너스 누적 시 100 초과 가능.
- 피터공: "상한 두지 말고 계속 해보자."
- 적용: score 상한 처리 없음. 누적 그대로.

**1.2 시나리오4 C2R3 = A 등급(70점) 유지**
- 발견: 룰 C2 = 35~45 인데 시나리오4 C2R3 = 70 (A 등급). "타인 의견 분리 카드"의 보상 디자인이지만 C 분기에서 A 등급이 의도와 어긋나 보일 수 있음.
- 피터공: "플테에서 여러 상황을 검증해 볼 수 있는 것에 무엇이 좋을지를 기준으로 아리공이 판단."
- 아리공 판단: 유지. 이유 — 시나리오4 진로의 "부모 의견 따라가다 R3 검수에서 자기 분리"는 진짜 학습 포인트. "C 분기여도 R3까지 가면 회수 가능"이 PX 메시지로 작동.
- 플테 체크포인트: 학생이 의도적으로 C 분기를 노리는 행동이 발견되는지 관찰.

**1.3 timeBudgetSec 데드 필드 — v0.6 정돈 사이클로 미룸**
- 발견: 4 시나리오 모두 `timeBudgetSec: 720`. v04/index.html grep 결과 게임 로직에서 미참조.
- 피터공 의도(레벨업 시 시간 증가): 이미 `gameState.resources.time.max` + `meterMaxByLevel`로 구현. timeBudgetSec와 무관.
- 결정: 그대로 두고 v0.6 정돈 사이클에서 SPEC 정리.

#### 2. 시나리오 선택 UI — 옵션 C 하이브리드 (4/28 세션251)

**2.1 시나리오 노출 정책**
- 후보 A: 시작 화면에 5장 카드 (자유 선택)
- 후보 B: 학기 순서 강제 (1만 활성, 클리어 시 2 열림)
- 후보 C: 하이브리드 — 5장 모두 노출 + 다음 추천 강조 + 자유 선택
- 피터공: "C로 만들면 플테할 때 순서대로, 또 랜덤하게 플레이가 가능하겠네 그렇게 하자."
- 적용: 5 시나리오 카드 + "다음 추천 →" 강조 (`.scenario-card.next`) + 다른 시나리오도 클릭 가능.

**2.2 시나리오 1회만 시도 가능**
- 피터공: "시나리오는 일단은 일회만 시도 가능하도록."
- 적용: `gameState.clearedScenarios` 배열 신설. 클리어한 시나리오는 카드 disabled (line-through + cursor:default).
- 영향: PLAN의 "다시 도전하기"(누적 유지) 버튼은 의미 변경 → "시나리오 선택으로"로 대체. `scenarioRepeatCount`는 사실상 0~1만 가능, 누적 메커닉(D-07)은 v0.6 이후로 미룸.

**2.3 학기 진행 프로그레스 UI**
- 피터공: "진행 프로그레스가 잘 인식되도록 UI를 만들어야겠는데."
- 적용:
  - 학기 진행 X/5 카운트 + 진행 바 + 5 dots (cleared/next/empty 상태)
  - 다음 추천 시나리오 자동 강조
  - 5 클리어 시 "한 학기를 모두 통과했다" 박스 노출

#### 3. 학기 캐릭터 누적 디자인 (암묵 결정, 4/28 세션251)

**3.1 시나리오 간 보존되는 상태**
- gameState.resources (시간/에너지) — 시나리오 종료 시 회복(recoverResources), 다음 시나리오 시작 시 그 자원으로
- gameState.exp + level — 학기 누적 (한 단계씩 올라감)
- gameState.competencies (delegationChoice + knowledge) — 누적
- gameState.cardsHeld + itemsCollected — 누적
- gameState.scenarioHistory — 시나리오별 결과 누적 (학기 종합 리포트 데이터)
- gameState.clearedScenarios — 클리어 시나리오 ID 배열

**3.2 시나리오 진입 시 리셋되는 것**
- currentScenarioId, currentTier, selectedTier1/2/Review, score, completed 플래그

**근거**: 같은 주인공의 한 학기. 다섯 결정의 누적이 그 학생을 만든다는 SCENARIO_PROMPT 메시지 구현.

#### 4. 활동 리포트 흐름 — 컷6 자체로 + 종합 리포트 분리 (4/28 세션251)

**4.1 매 시나리오: 컷6 자체가 결과 화면**
- 피터공: "활동리포트는 5개 다 하면 마지막에 종합해서 보여주는 것으로." + "지금도 활동리포트로 빠지는데."
- 변경 전: 컷6 진행 → "활동 리포트 보기" 버튼 → showReport 별도 화면
- 변경 후: 컷6 패널에 이미 점수/등급/아이템/awareness/이번 회기 영향/누적/exp/LV UP/자원 회복 다 표시. "활동 리포트 보기" 버튼 → "다음 시나리오로 →" 또는 "학기 종합 리포트 보기"로 변경. showReport 화면 거치지 않음 (함수는 코드에 잔존, 호출처 없음).

**4.2 5 시나리오 다 끝나면: 학기 종합 리포트 (`showFinalReport`)**
- 헤더: "한 학기 활동 리포트" + Arc 안내 한 줄
- 통계 6칸: 학기 총점 / 평균 / 최종 레벨 / 위임 선택력 / 지식 / 카드·아이템 수
- "다섯 결정의 발자국": 시나리오별 (제목 / leaf / 점수 / 등급 / 아이템 + awareness 인용 + learningMessage)
- 학기 회고 한 줄 (평균 점수 기반 4단계 톤 — 80+/60+/40+/40-)
- "학기 처음부터" 버튼

#### 5. 레벨업 메커닉 — 단계별 + exp 리셋 (4/28 세션251)

**5.1 한 번에 한 단계만 + 잉여 exp 손실**
- 피터공: "Lv 버그는 한번에 값이 커져서 그런 것 같은데 그 경우도 단계별로 올라가야 할듯. 다음 단계로 올라가면 경험 포인트는 다시 리셋되고 카운트 시작."
- 변경 전: `checkLevelUp` 다중 레벨업 (한 번에 Lv1→Lv3 가능)
- 변경 후: 한 단계만 진입. 진입 시 `exp.current = thr[lv-1]` (clamp) → 단계 안 진행 0부터 카운트
- 표시: "+45 exp → Lv.2 0/30 (다음 단계까지)"
- 부수효과: 잉여 exp 손실 (예: Lv1+45 → Lv2 진입, 25 exp 손실)

**5.2 expThresholds 조정 [0,20,50,100,200] → [0,20,40,65,100]**
- 피터공: "임계를 조금 조정해서 최고 점수 선택들이 이어지면 현재 학기 동안도 Lv5 할 수 있게 해보자."
- 단계 임계 변화: 20 / 30 / 50 / 100 → 20 / 20 / 25 / 35
- 시뮬 결과 (S 등급 +45 exp × 5 시나리오):
  - 변경 전: S5 후 Lv4 머무름 (Lv5 도달 X)
  - 변경 후: S4에서 Lv5 도달 ✓ (S5는 머무름)
- 평범 mix (A1R3=30, A2R3=23, B3R3=14, B2R3=12, A3R3=17): Lv4 도달
- 위임/우회 위주(B1, C 분기): Lv2~3
- 디자인 의도: 최고 선택 누적 → 학기 안에 Lv5 도달 가능 (가시적 보상)
