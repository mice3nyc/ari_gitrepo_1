#### 3. v0.4 기획 Q&A 9개 (4/28 18:00~18:35)

##### Q1. 시간 vs 에너지의 차이

**아리공 가설**: 시간 = 거시 자원 (시나리오 데드라인) / 에너지 = 미시 자원 (의지/집중력)

**피터공 답**:
- 시간 = 작업에 걸리는 시간. AI 없이 하면 시간이 다 소진하고도 부족할 수도 있다.
- 에너지 = 노력/수고로움. **지난한 수련의 시간!** 실제 공부 역량 강화는 이를 통한 경험치 증가가 핵심.
- 자원 조합 다양: 시간 낮+에너지 높 / 시간 높+에너지 높 / 시간 높+에너지 낮 등 다양한 조합 가능.

> **핵심 메커닉**: 에너지 = 경험치 변환기. 시간 절약 위해 AI 위임만 하면 에너지 안 쓰니 경험치 안 쌓임 → 레벨업 X → AI 검수 역량 안 늚 → 악순환. **이게 메시지의 정확한 설계점.**

##### Q2. 자원 회복 메커니즘

**피터공 답**:
- 시나리오 종료 시 일부 회복 + **결과물 퀄리티에 따라 회복량 차등**
- 소진/획득 흐름이 게임 루프. 너무 쉽게 다 회복은 X.

**Q6 후속**:
- 기본 70% 회복 (시간/에너지 모두)
- 결과 등급에 따라 +10~60% 추가 → 최저 ~80% / 최고 ~130%(미터 max에 닿음)
- 잘했을수록 기본 보충에 잉여까지

##### Q3. 학습 곡선

**피터공 답**:
- 초반: AI 위임 → 시간 ↓ but 결과 퀄 ↓ (검수 약함)
- 초반에 자기개발/학습(에너지 쓰는 직접 작업)에 투자 → 후반: AI 협업으로 시간 절약 + 완성도 ↑
- **시나리오가 이 학습 곡선에 맞게 순차적으로 설계되어야 함** (v0.5 과제)

##### Q4. 도메인 역량

**피터공 답**:
- 일단 하나로 시작 (= v0.3의 `knowledge` 그대로)
- 여러 역량 가능성 열어두되, 시나리오별로 다른 역량 강화 영향 가능 (v0.5 이후)
- **혼돈 X, 직관 유지**

##### Q5. 힌트 숨김

**피터공 답**:
- **힌트 토글 버튼** 도입. 테스트 가능
- 처음엔 OFF (= 기본 숨김)
- v0.3 현재: 선택지에 영향 미리 표시 → v0.4: 숨김

##### Q6. 결과물 퀄리티 → 회복량 (Q2 후속)

**아리공 가설**: v0.3 5등급(S/A/B/C/D)을 그대로 환산기로

**피터공 답**: 그렇게 하자. 기본 70% + 등급 보너스 10~60%

→ **5등급 환산 초안 (아리공 제안)**: D=+10, C=+22, B=+34, A=+47, S=+60 (선형 분배)

##### Q7. 한 시나리오 = 한 자원 사이클?

**피터공 답**: YES.
- 1회 플레이(6단계) = 시간/에너지 소비 → 결과 → 회복 → 다음 시나리오 시작
- 시나리오 안에서는 회복 X
- 종료 시에만 회복

##### Q8. 경험치 → 레벨 단계

**피터공 답**:
- 시나리오 중에는 레벨업 발생 X (단순화). **시나리오 종료 시에만 레벨 업데이트**
- 일단 **5단계** 시작
- 같은 시나리오 반복 시 경험치 누적 (성장 가능)
- **레벨업 → 시간/에너지 미터(최대치) ↑** (새 메커닉)
- **레벨업 직후 = 시간/에너지 꽉 찬 상태로 다음 시나리오 시작** (큰 인센티브)

##### Q9. v0.4 스코프

**아리공 권장**: (a) 어린왕자 1 시나리오 + 자원 시스템만 = MVP. 학습 곡선 시나리오 3개는 v0.5.

**피터공 답**: "일단 (a)로 시작해서 기능 확인! 너무 크니까!" ✓
