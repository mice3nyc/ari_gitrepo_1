## 층위 구조 — 이 순서를 지킨다

1. **규칙 층** — 섞인 경로를 어떻게 평가하는가 (합성 규칙)
2. **데이터 층** — 135 결말 수기 값
3. **검사 층** — 린터(정합성확인시스템): 규칙 vs 데이터 자동 대조
4. **밸런스 층** — 수치 튜닝 (S1·5분면 비용·갈래 2)

> 핵심 원칙: **논리 정합이 먼저, 밸런스는 그 위.** 규칙 없이 수치를 조이면 모순 위에 튜닝하는 것.
> 카드도 마찬가지: **분포 중심이 아니라 논리 중심** — 카드는 그 결말 경로가 실제 보인 행동의 증거로 도출하고, 분포(종별 개수)는 결과로 따라오는 검수 지표일 뿐.

## Phase 0 — 피터공 결정 게이트 ✅ (6/10 오후 확정)

- [x] **D-1. 카드 지급 원칙 = 행동(선택) 기준.** 피터공: "좋은 선택이 역량카드도 많이 생기는 것이 옳다. 등급이 낮을수록 카드 얻을 선택을 안 했을 것." → 원리: **카드는 결말이 아니라 선택에서 나온다.** 등급도 같은 선택에서 나오므로 "등급↓=카드↓" 상관은 규칙 없이 자연 성립. 6/19 구현: 결말(leaf) 카드 값은 유지하되 **린터가 "경로의 선택 행동 ↔ 카드"를 대조**해 어긋난 곳만 정비. 카드를 선택지 데이터에 직접 부착하는 구조 전환은 KT 후 과제.
- [x] **D-2. 검토 차등 채택.** "셋이 같이 오르는 건 말이 안 됨"(피터공). 세 활동 = 그냥 넘기기 / 조금 손보기 / 정말 검토하기(대사는 브랜칭별 달라도 활동은 셋). 수치 스킴(안: 넘기기 0 / 손보기 절반 / 검토 온전)은 정비 단계에서 표로 결정. **+ 추가 방향(피터공)**: 어떤 검토를 하는지가 브랜칭별로 **다른 역량카드 획득**으로 이어지는 게 더 적합 — 검토 선택지의 discountTags(R2 검토력·표현력 / R3 검토력·자기이해)가 이미 그 매핑을 갖고 있어 자연스러운 첫 적용 지점. D-1의 "카드는 선택에" 원리와 한 몸.
- [x] **D-3. 6/19 범위 = 규칙 문서화 + 자동 검사기 + 모순 정비.** 비용·할인 재설계는 KT 후 패키지.

### Phase 0 추가 결정 — UI/UX 방향 (6/10 피터공)

- **"체감되도록 만들자! 크기와 가시성!"** — 할인 작동 기제와 역량 미터(두 힘 게이지) 두 가지를 UI/UX로 업데이트. 두 수치의 기능(할인)은 옳되 체감·가시성이 부족하다는 진단에 합의. S3 표현 변경(수정안 ①~④)과 같은 트랙.
- **(차기 정리) 할인 전체를 역량카드의 게임 아이템 기능으로**: 현재 에너지 할인은 "아는것의 힘 수치(자동·근거 비표시) + 쿠폰 카드 한 장(−2/−3·카드명 표시)" 이중 구조인데, 이걸 전부 카드 아이템 작동으로 통합하면 더 직관적이고 역량 획득이 성취감이 됨 — 피터공 "나중에 따로 정리하자". 두 힘-두 트랙 메커니즘화(인간중심→시간/도메인→에너지)와 같은 자리.
