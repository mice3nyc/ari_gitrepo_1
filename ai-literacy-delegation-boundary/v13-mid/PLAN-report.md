## 목적

핵심 명제: **좋은 리포트는 점수를 보여주는 것이 아니라, 학생이 자기 플레이를 자기 말로 다시 설명할 수 있게 하는 기록이다.** 사후수업은 교사가 리포트를 해설하는 시간이 아니라 학생이 리포트를 들고 말하는 시간. 등급은 trade-off의 거울 (6/9 S2 합의의 구현).

해결하는 문제 둘:
1. 리포트가 6/11 폐지된 옛 모델(±수치·4분면 유형·legacy competencyCards)로 말하고 있다 — 게임 본편과 언어 불일치
2. 교사 피드백 S2 "등급 기준이 안 읽힌다" + "성찰 강화" — 질문을 붙이지 않고 기록의 형태로 푼다

## 피터공 질문 프로토콜 (이 트랙의 운영 규칙)

피터공이 출장 중이라 자세한 중간 검토 불가. 합의된 방식:

- **질문은 라운드 게이트에서만, 묶어서 1회.** 작업 중간에 낱개로 묻지 않는다
- **형식: 한 줄 질문 + 보기 2~3개(추천 표시).** 툭 보고 탁 답할 수 있게
- **답이 늦으면 추천안으로 진행**하고 라이브 빌드에서 뒤집을 수 있게 만든다 (되돌리기 쉬운 구현 우선)
- 결정이 아닌 것(구현 디테일·문구 다듬기)은 아리공 판단으로 진행, 결과만 보고

### R3.5 — 항로 v2: 마이크로 항로 + 만화 타임라인 통합 (피터공 6/12 방향 승인, 다음 회기 착수)
피터공 라이브 확인 피드백: "위임 항로 아주 마음에 든다" + 확장 둘.
1. **시나리오 내부 세부 항로**: 시나리오 안의 세 비트(1차 선택→2차 선택→검토)가 항로에 보이게. 데이터는 history에 전부 있음. **열린 설계 결정: tier2의 위임 스펙트럼 가로 위치 매핑 규칙** (시안 만들어 3차 게이트)
2. **만화 타임라인 통합**: 하단 독립 만화 섹션을 폐지하고, 항로 옆에 해당 장면 컷이 붙는 세로 타임라인으로. **가독성 리스크**: 행당 컷 5장+항로 동거 시 기울기 가독성 — 해법 후보 = 대표 컷 1~2장(클릭 확장) 또는 좌항로·우만화 2단 정렬 (3차 게이트에서 시안 비교)
- 착수 절차: SPEC §4d 선문후코 → 시안 게이트(한 줄+보기) → 구현. R4(영수증)보다 앞 순서로 당김

### R3.6 — 항로 비용 열 + AI 표시 (피터공 6/12 제안, 같은 날 구현)
항로 행마다 오른쪽 고정 열에 1차·2차·검토 각각의 실지불 시간·에너지(R2 discounts 합산)를 세워, 다섯 행을 세로로 훑으면 위임·직접의 가격 차가 보이게. 1차·2차 선택에 AI 개입 칩(ai_flags.yaml 수기 태깅 60개, micro_offsets 패턴). SPEC §4e.
