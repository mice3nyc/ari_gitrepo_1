## 1. 지급 규칙 v0 (데이터 기반 도출)

선택지 노드에 카드 필드를 신설하지 않고, 기존 discountTags + delta 부호에서 도출한다.

| 단계 | 조건 | 지급 카드 |
|---|---|---|
| tier1 | `getAxisDelta(delegation) > 0` | 인간중심 — 축=`discountTags.humanCentric`, 태그=축→태그 맵 |
| tier2 | `getLeafDelta(t2, t1).knowledge`가 `+`/`++` | 도메인 — `discountTags.strongDomain[0]` |
| tier2 | 위와 별도로 `delta.delegation === '++'` | 인간중심 추가 (이번 시나리오에 같은 축 없을 때만) |
| review | `discountTags.strongDomain` 비어있지 않음 (R2/R3) | 도메인 — `strongDomain[0]` |
| 결말 | 등급 B/C/D → 회복력, 리플레이 완료 → 도전력 | **기존 유지** (등급 기반은 끝이 맞음) |

**쿠폰 사용 제한 (6/11 피터공 결정)**: 선택별로 획득한 카드(`perChoice:true`)는 **획득한 그 시나리오 안에서는 할인 쿠폰으로 쓸 수 없다**. 시나리오를 마치면 다음 시나리오부터 작동. 구현: 쿠폰 가용 카드 계산(04-resources `_calculateCardEnergyDiscount`)에서 `perChoice && scenario === currentScenarioId` 항목 제외. (같은 시나리오 리플레이에서도 제외됨 — "다음 시나리오부터"의 보수적 해석.)

축→태그 맵 (selfintro): 중심잡기→주체성, 융합하기→통합적 사고, 성찰하기→성찰적 사고.

## 2b. 획득 연출 v1 — 우측 팝업 + 카드 레일 (6/11 2차, 피터공)

> "선택 버튼을 눌러서 획득이 되면 우측에 팝업으로 '○○○ 선택 → ○○ 능력 획득!' 안내가 뜨고, 창을 닫으면 우측 빈 공간에 카드가 팝! 하며 등장. 쌓인 카드들은 시나리오 전체가 종료되면 역량카드 버튼으로 날아들어가 사라진다."

기존 전면 오버레이 토스트(`playCardRewardSequential`)를 파일럿 시나리오에서 **우측 팝업 + 카드 레일**로 교체한다.

- **획득 팝업** (`showCardEarnPopup(choiceLabel, cards)`): 화면 우측 고정(레일 위쪽). 1줄 "「선택 라벨」 선택" + 큰 글씨 "○○ 획득!" + 카드 미리보기(축 색/표시명). 닫기 = X 버튼·팝업 클릭, **4초 후 자동 닫힘 병행**. 닫히면 카드가 레일로 팝 등장(여러 장이면 0.15s 시간차).
- **카드 레일** (`#card-rail`): position:fixed 우측 가장자리, HUD 아래에서 시작, 획득 순서대로 위→아래 스택. 미니 카드(라벨+축/도메인 색 스트라이프). 시나리오 시작·리플레이 시작 시 비움(`railClear()`).
- **종료 비행** (`railFlyToInventory()`): 컷6 체인 끝(모달들 이후, 다음 버튼 노출 직전)에 레일 카드들이 역량카드 버튼(`#inv-tab`)으로 순차 비행(기존 `travelCard` 곡선 재사용) + 탭 펄스. 레일 비면 no-op — 비파일럿 시나리오에 영향 없음.
- **검토 카드 시점 이동**: 컷6 체인의 검토 카드 적립·표시를 `onReview` fadeOut 콜백으로 이동 — tier1·tier2와 동일하게 "선택 직후" 한 흐름. 컷6 체인의 파일럿 분기는 제거(도전력·회복력은 기존 위치 유지).
- **신설 파일** `src/js/16-card-rail.js` + `08-inventory-and-rewards.css`에 레일/팝업 스타일. 비파일럿 시나리오의 결말 일괄 토스트는 그대로(Phase 2 전체 개편 때 통일).

## 2c. 획득 연출 v2 — 팝업을 컷 이미지 위로 + 레일까지 비행 (6/11 세션466, 피터공)

> "팝업 윈도우가 눈에 잘 안 띈다. 지금 선택을 한 시나리오 cut 이미지 보여주는 위치 이미지 위에 띄워주고, 거기서 카드가 우측으로 날아가는 거지."

- **팝업 위치 변경**: 우측 고정 → **방금 선택으로 활성화된 컷의 panel-image 위 오버레이**. 앵커 컷 = stage 매핑 {tier1→컷2, tier2→컷3, review→컷5}(선택 요약이 뜨는 컷). `showCardEarnPopup(label, cards, anchorCut)` 3번째 인자 추가. position:absolute(문서 좌표)라 스크롤 중에도 이미지에 붙어 있음. 등장은 scale 0.85→1(`.over-image`). 앵커를 못 찾으면 기존 우측 고정 폴백.
- **닫힘 = 우측 비행**: 팝업이 닫힐 때 미리보기 카드들이 그 자리에서 **고스트로 떠 우측 레일 슬롯으로 비행**(0.5s, 장당 0.15s 시간차) → 도착 시 레일에 팝 등장(railAddCard). 기존 "팝업 닫힘 → 레일에서 갑자기 팝"의 끊긴 연출을 잇는다.
- 종료 비행(레일→역량카드 버튼)·4초 자동 닫힘·railFlyToInventory의 선닫힘 가드는 무변.
- **v2.1 보강 (피터공)**: ① 등장 = "띠용 팝" — scale 0.3 → 1 오버슈트 바운스(0.45s, cubic-bezier(0.34,1.56,0.64,1)) ② 위치 = 컷 이미지 **가로 중앙 + 하단부**(이미지 바닥에서 14px 위, 팝업 높이만큼 끌어올림) ③ 크기 업 — over-image 모드 width 300px(≤1320px 260px)·padding 20/24·획득 글자 18px.

## 2d. 카드 독 — 우측 상시 컬렉션 + 회전 비행 + 철컥 고정 (6/11 세션466, 피터공)

> "역량카드가 애매한 위치의 버튼을 누르면 열리는 창이 아니라, 화면 우측 끝에 위에서부터 획득 순서대로 쌓이는 것. 큰 순서는 인간중심역량카드 그리고 능력카드. 그 틀 안에 획득된 카드가 회전해서 날아 들어가고, 시나리오가 완료되면 완전히 획득한 모양으로 철컥하고 고정."

- **#card-dock**: 화면 우측 끝 고정(HUD 아래~바닥, overflow-y 스크롤·스크롤바 숨김). 섹션 2개 고정 순서 — **인간중심 역량** / **능력 카드**(도메인+성장[회복력·도전력]은 능력 하단 — 피터공 확인 포인트). 섹션 안은 획득 순서(인벤토리 배열 순). 표시는 게임 화면(showStats)과 동기, 타이틀·리포트에선 숨김.
- **기존 요소 대체**: 우측 중앙 `inv-tab` 버튼 숨김(§2b 레일 #card-rail·종료 비행 폐기). 독 클릭 = 기존 상세 패널(inv-panel) 열기 — "처음부터 다시" 풋터 접근 유지. 팝업 미리보기용 `.rail-card` 스타일은 존치.
- **상태 2단**: 현재 시나리오에서 선택으로 획득한 카드(perChoice & scenario=현재 & 미클리어) = **임시(pending)** — 점선 테두리·살짝 흐림. 시나리오 완료(컷6 체인 끝, 구 railFlyToInventory 자리)에 **철컥(locking)** — 카드별 120ms 시간차로 쾅 찍히는 스냅 애니메이션 후 실선+그림자 고정(locked). 이전 시나리오 카드는 처음부터 locked로 렌더.
- **회전 비행**: 획득 팝업 닫힘 → 독에 pending 칩 자리 생성(숨김) → 팝업 미리보기 카드 고스트가 **rotate(540°)** 돌며 칩 자리로 비행(0.55s) → 도착 시 칩 pop 등장.
- **데이터 무변**: 인벤토리 구조(humanCentricCards/domainCards/growthCards) 그대로, 독은 읽기 전용 미러. 호환: `railClear`=독 재렌더, `railFlyToInventory`=철컥(이름 유지, 호출처 무변).
- **§2c 위치 버그 수정 동반**: 팝업 앵커를 getBoundingClientRect → **offsetLeft 체인**(문서 좌표)으로 — 컷 패널 slide-in(translateX 30px→0) 도중 측정돼 우측으로 밀리던 문제.
- **v2 보강 (피터공 5건)**: ① inv-tab 버튼·inv-panel 상세 패널 완전 폐지(독 클릭 동작 제거. "처음부터 다시" 리셋은 디버그 패널 초기화로만) ② 섹션 레이블 10→13px ③ 칩 한 줄 표기("중심잡기 주체성" — axis+tag 인라인, nowrap) + 독 폭 158→200px(≤1320px 170px) ④ **pending = 진한 회색 글씨·컬러코딩 없음·점선 테두리·깜빡임**(dockBlink 1s step). 컬러(축 배경/도메인 스트라이프)는 철컥 고정 때 입힘 ⑤ 칩 글씨 12→14px.

## 2e. 획득 팝업 v3 — 확보 버튼 + 카드명 상단 큰 글씨 (6/11 세션468, 피터공)

## 5. Phase 2 — 전체 적용 (6/11)

- **대상**: 5종 전체 (`selfintro, groupwork, eorinwangja, career, studyplan`). 지급 규칙은 §1 그대로, 시나리오별 축→태그 맵만 추가.
- **축→태그 맵 선정 원칙**: selfintro = 파일럿 승인 맵 유지. 나머지 4종 = finals(v22 정비분) 분포에서 **최빈 태그, 동률이면 최고 등급(S) 경로의 태그**. 시나리오 정체성과 부합(모둠→사회·관계, 진로→창의, 시험→문제해결). **+ 6/11 피터공 "시나리오별로 다른 태그가 좋을듯, 5번 모두 주체성은 좀"** → 중심잡기를 시나리오별 분산(중심잡기 축 태그는 주체성·적응성·호기심 3종뿐이라 일부 겹침은 허용).

| 시나리오 | 중심잡기 | 융합하기 | 성찰하기 |
|---|---|---|---|
| selfintro | 주체성 | 통합적 사고 | 성찰적 사고 |
| groupwork | 적응성 (모둠 호흡) | 통합적 사고 | 사회·관계적 사고 |
| eorinwangja | 호기심 (책에 질문) | 맥락적 사고 | 윤리적 사고 |
| career | 호기심 (진로 탐색) | 창의적 사고 | 비판적 사고 |
| studyplan | 주체성 (스스로 계획) | 문제해결적 사고 | 성찰적 사고 |

## §6. 카드 표기 — 축 이름 제거 (6/12 피터공 결정)
