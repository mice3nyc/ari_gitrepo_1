## 목적

중등 「위임의 경계」를 멋지게 완성한 코드 위에, **데이터만 초등으로 갈아끼워** 초등 버전을 만든다. 코드를 새로 짜지 않는다. 중등 배포본은 안전하게 둔다.

## 설계 원칙 (피터공 확정 6/21)

1. **분리 빌드**: `v13-elem/`은 중등 코드 복사본. v13-mid 무수정.
2. **데이터만 교체**: 선택지·결과는 초등 브랜칭 CSV. 빠진 콘텐츠(desc·lesson·캡션·리포트)는 docx 기반 아리공 저작.
3. **카드 = 중등과 동일 세트**: 능력카드·역량카드 모두 중등 카드. 회복력만 적응성 교체. CSV가 선택마다 지정한 카드를 그대로 지급(명시 per-choice, SPEC-elem-variant.md §5).
4. **엔진은 가능한 한 그대로**: 데이터가 요구하는 최소 지점만 v13-elem 안에서 손봄(§4 핫스팟).
