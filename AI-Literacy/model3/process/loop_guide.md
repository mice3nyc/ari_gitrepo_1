# Loop Guide — 에이전트 자율 루프 진행 가이드

> 이 문서는 에이전트가 매 단계마다 따르는 체크리스트이자 절차서다.
> 루프를 도는 에이전트는 반드시 이 파일을 먼저 읽고, 단계별로 체크하면서 진행한다.
> 이 가이드 자체도 루프 안에서 갱신될 수 있다.

---

## 0. 루프 시작 전 — 자기 위치 파악

에이전트가 루프를 시작할 때 반드시:

- [ ] `manifest.yml` 읽기 — 프로젝트 현재 phase, 각 파일의 status 확인
- [ ] `state/scratch_pad.md` 읽기 — 지금 어디까지 왔는지 확인
- [ ] 마지막 `runs/run_N.md` 읽기 — 직전 라운드에서 뭘 했고 뭘 바꾸기로 했는지
- [ ] canon/ 파일들 읽기 — 현재 기준 확인 (canon은 갱신될 수 있음)

---

## 1. PROBE 단계 — 본문(전체 시나리오) 전 검증

전체를 만들기 전에 **조각을 먼저 만들어 검증한다.**

### Probe란?
- 시나리오의 핵심 장면 1개, 또는 캐릭터 소개 1개, 또는 선택지 1세트
- **작은 조각**을 canon 기준으로 평가하는 것

### Probe 절차
1. probe 작성 → `probes/probe_N.md`에 저장
2. constitution.md의 **필터(B섹션)**로 검증:
   - [ ] 교훈 착지가 없는가?
   - [ ] 정답으로 수렴하지 않는가?
   - [ ] "부담스러울 정도로 현실적"인가?
   - [ ] 선택에 trade-off가 있는가?
   - [ ] 초중등이 상상할 수 있는 수준인가?
   - [ ] "밀어내야 할 전개 운동"에 해당하지 않는가?
   - [ ] "밀어내야 할 문장 운동"에 해당하지 않는가?
3. **3개 이상 통과 → 본문 진입 가능**
4. 통과 못 하면 → 수정 후 재검증, 또는 canon 갱신 필요 여부 판단
5. 검증 결과를 probe 파일에 기록

---

## 2. RUN 단계 — 라운드 실행

### 한 라운드에서 하는 것
1. **목표 설정**: 이번 라운드에서 뭘 만들/바꿀 것인가
2. **실행**: 콘텐츠 작성 또는 수정
3. **자체 평가**: constitution + completion 기준으로 평가
4. **canon 갱신 여부 판단**:
   - 이번 라운드에서 새로 발견한 "밀어내야 할 것"이 있는가?
   - constitution에 추가할 원칙이 있는가?
   - 있으면 → canon 파일 직접 갱신 + 갱신 이유 기록
5. **기록**: `runs/run_N.md`에 전부 기록

### Run 기록 형식 (runs/run_N.md)
```markdown
# Run N — [제목]

## 목표
이번 라운드에서 하려는 것

## 실행 내용
실제로 한 것 (콘텐츠/수정 내용)

## 자체 평가
### Constitution 필터 체크
- [ ] 교훈 착지 없음
- [ ] 정답 수렴 없음
- [ ] 현실적 부담감 있음
- [ ] Trade-off 있음
- [ ] 초중등 상상 가능
- [ ] 밀어내야 할 전개 운동 해당 없음
- [ ] 밀어내야 할 문장 운동 해당 없음

### 통과/실패 판정
- 통과: [항목]
- 조건부: [항목 + 이유]
- 실패: [항목 + 이유]

## Canon 갱신
- 갱신함 / 갱신 안 함
- (갱신 시) 어떤 파일의 어떤 부분을 왜 바꿨는가

## 다음 라운드 계획
- 다음에 바꿀 것
- 다음에 시도할 것

## State 갱신
- scratch_pad.md 업데이트 내용
```

---

## 3. REVIEW 단계 — 라운드 간 점검

매 3라운드마다 (또는 큰 방향 전환 시):

- [ ] 전체 runs/ 훑어보기 — 같은 문제가 반복되는가?
- [ ] canon/ 변경 이력 확인 — 기준이 어떻게 성장했는가?
- [ ] scratch_pad 정확한가 — 현재 상태가 실제와 일치하는가?
- [ ] manifest.yml 갱신 필요한가?

---

## 4. FINAL 단계 — 피터공에게 제출

### 제출 조건
- 최소 3라운드 이상 진행
- 마지막 라운드에서 constitution 필터 전항목 통과
- **최종 후보 2개** 준비 (하나만 내지 않음)

### 제출물
1. 최종 후보 2개 (콘텐츠/시나리오)
2. 각 후보의 장단점 비교
3. runs/ 전체 (과정 기록)
4. canon/ 최종 상태 (기준이 어떻게 성장했는지)

---

## 핵심 원칙

1. **Canon은 고정이 아니다.** 루프를 돌면서 기준 자체가 성장한다. 발견한 것을 canon에 반영하라.
2. **Probes를 건너뛰지 마라.** 전체를 만들고 평가하면 수정 비용이 크다. 조각을 먼저 검증하라.
3. **기록 없는 라운드는 없다.** 모든 라운드를 runs/에 기록한다. "해봤는데 안 됐다"도 기록이다.
4. **"버리기"를 기록하라.** 뭘 시도했다가 왜 버렸는지가 다음 라운드의 필터가 된다.
5. **scratch_pad는 매 라운드 후 갱신.** "지금 어디에 있는가"가 항상 정확해야 한다.
