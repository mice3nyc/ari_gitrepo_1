# Arc — AI 리터러시 모델2 UX 흐름

> 플레이어가 실제로 하는 행동의 순서. 15~20분 완결.

---

## 전체 흐름

```
[인트로] → [스캔] → [수집+검증] → [조립] → [브리핑] → [거울]
  1분       2분      7~8분         3~4분     2분       1~2분
```

---

## Phase 1: 인트로 (1분)

### 목적
역할 부여 + 이슈 진입

### 플레이어 경험
- "당신은 시그널 에이전트입니다"
- 오늘의 미션 브리핑: "[AI가 그린 그림이 미술 대회에서 대상을 탔다] — 이 이슈에 대한 시그널 브리핑을 작성하세요"
- 4가지 신호 영역(기술/사회/개인/미래)이 빈 보드로 표시됨
- "신호를 수집하고, 노이즈를 걸러내고, 당신만의 브리핑을 완성하세요"

### 인터랙션
- 미션 수락 (한 번 클릭으로 시작)
- 보드가 열림

---

## Phase 2: 스캔 (2분)

### 목적
정보 카드 전체를 훑어보기. "뭐가 있는지 파악"

### 플레이어 경험
- 16~20개의 정보 카드가 피드/대시보드 형태로 나열됨
- 각 카드: 제목 + 신호 유형 컬러 + 출처 한 줄
- 스크롤 또는 스와이프로 훑어봄
- 아직 수집하지 않음 — 전체 그림을 먼저 파악

### 인터랙션
- 카드 탭하면 내용 펼쳐짐 (2~3문장)
- 관심 있는 카드에 "핀" 찍기 가능 (나중에 돌아오기 쉽게)

---

## Phase 3: 수집 + 검증 (7~8분) ← 코어 게임플레이

### 목적
4가지 신호를 수집하되, 각 카드의 신뢰도를 판단해야 함

### 플레이어 경험

**수집 행위:**
- 카드를 보드의 해당 신호 영역으로 드래그 (또는 탭→배치)
- 각 영역에 최소 1개, 최대 3개 카드 배치 가능

**검증 관문 (카드를 보드에 올리는 순간):**
- "이 정보를 신뢰합니까?" 판단 요청
- 출처 정보 + 교차 정보 제공
- 학생이 판단: ✅ 시그널로 수집 / ⚠️ 노이즈로 제외 / 🔍 보류
- **노이즈를 시그널로 올리면**: 브리핑에 경고 마크 (페널티가 아니라, "이 신호는 검증이 약합니다" 표시)
- **시그널을 노이즈로 버리면**: 놓친 신호로 기록 (브리핑 후에 "이것도 있었습니다" 표시)

**개인 신호 (🔴):**
- 텍스트 카드가 아닌 **선택형 조각**
- "이 이슈가 나에게 의미하는 건 ___이다" → 3개 조각 중 선택
- 예: "수행평가 기준이 바뀔 것이다" / "내 취미의 가치가 달라진다" / "별로 상관없다"

**미래 신호 (🟣):**
- **시나리오 조립**
- 3개의 시나리오 조각이 제공됨, 2개를 선택해서 연결
- 예: [AI 아트가 주류가 된다] + [사람 그림이 프리미엄이 된다] → 학생이 연결문 선택
- 조합에 따라 다른 미래 시나리오가 생성됨

### 인터랙션
- 드래그 앤 드롭 (또는 탭→배치)
- 검증 판단 버튼 (✅/⚠️/🔍)
- 개인 신호: 선택지 탭
- 미래 신호: 조각 선택 → 연결

---

## Phase 4: 조립 (3~4분)

### 목적
수집한 신호들을 배치하고 비중을 정해서 브리핑 구조를 완성

### 플레이어 경험
- 보드에 올린 신호들이 나열됨
- **비중 설정**: "이 이슈에서 가장 중요한 신호는?" → 하나를 메인으로 선택
- **연결**: 신호 사이의 관계를 선택 (화살표로 "이것이 이것에 영향을 준다")
- **헤드라인**: 내 브리핑의 한 줄 제목을 3개 후보 중 선택 (또는 직접 입력)

### 인터랙션
- 메인 신호 선택 (탭)
- 신호 간 연결선 긋기 (드래그)
- 헤드라인 선택 (탭 또는 타이핑)

---

## Phase 5: 브리핑 (2분)

### 목적
완성된 브리핑을 시각적으로 보여줌

### 플레이어 경험
- 내가 구성한 브리핑이 **한 장의 카드/리포트**로 생성됨
- 구성: 헤드라인 + 4가지 신호 요약 + 메인 신호 강조 + 미래 시나리오
- 검증 약한 신호에는 ⚠️ 마크
- "당신의 브리핑은 [기술 중심형 / 사회 관찰형 / 개인 체감형 / 미래 설계형]입니다"

### 인터랙션
- 브리핑 생성 애니메이션 (보상감)
- 공유 버튼 (스크린샷/링크)

---

## Phase 6: 거울 (1~2분)

### 목적
"너는 이 이슈를 이렇게 보는 사람이다" — 발견

### 플레이어 경험
- **프로필 유형**: 어떤 신호를 강조했는지에 따른 유형
  - 🔵 기술 중심형: "변화의 속도를 먼저 본다"
  - 🟡 사회 관찰형: "사람들의 반응을 먼저 본다"
  - 🔴 개인 체감형: "나에게 닿는 것을 먼저 본다"
  - 🟣 미래 설계형: "앞으로 어디로 가는지를 먼저 본다"
- **놓친 신호**: "이런 신호도 있었습니다" (선택하지 않은 카드 중 중요한 것)
- **다른 학생은?**: 같은 이슈에서 다른 학생들의 브리핑 유형 분포 (교실 단위)
- **판정 없음**: 어떤 유형이 "좋다"는 말 없음. 모든 유형이 동등

### 인터랙션
- 프로필 카드 표시
- "다른 에피소드 해보기" 버튼
- "다시 하기" (다른 신호 선택으로)

---

## 교사 연계

게임이 끝난 뒤 교사의 영역:
- "같은 이슈에서 기술 중심형과 개인 체감형이 나왔다 — 왜 다르게 봤을까?"
- "노이즈로 버린 카드가 사실은 시그널이었다면? 어떻게 알 수 있었을까?"
- "내 브리핑에서 가장 자신 있는 부분과 불확실한 부분은?"
