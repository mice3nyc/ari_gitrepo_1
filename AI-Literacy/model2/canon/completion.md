# Completion — AI 리터러시 모델2 평가 기준

> 루프의 각 라운드와 최종 산출물을 판정하는 기준.
> Constitution의 D(선언)과 C(판정 기준)을 실행 가능한 체크리스트로 변환한 것.

---

## 1. 프로토타입 검증 (기본 — 매 라운드)

### 기술적 체크
- [ ] 단일 HTML 파일로 작동하는가 (프레임워크 없음, 바닐라 JS)
- [ ] 모바일(폰/태블릿)에서 레이아웃이 깨지지 않는가
- [ ] PC에서도 충분한 크기로 보이는가
- [ ] 15~20분 안에 전체 플로우를 완주할 수 있는가
- [ ] 인터넷 연결 없이 작동하는가 (오프라인 가능)

### 인터랙션 체크
- [ ] 카드 수집 → 검증 관문 → 보드 배치 흐름이 끊김 없는가
- [ ] 개인 신호 조각 선택이 "글쓰기"가 아니라 "탭"으로 되는가
- [ ] 미래 신호 조립이 2개 선택 → 연결로 직관적인가
- [ ] 드래그 앤 드롭이 모바일 터치에서 작동하는가

---

## 2. 콘텐츠 검증 (코어 — 매 라운드)

### 거울 테스트
- [ ] **같은 정보 더미에서 시작해서, 학생마다 다른 브리핑이 나오는가?**
  → 3명이 플레이했을 때 최소 2개 이상 다른 유형이 나와야 함
- [ ] 수집 과정에서 "내가 이걸 왜 골랐지?"를 느끼는 순간이 있는가?
  → 선택이 무의미하게 느껴지면 실패

### 검증 관문 테스트
- [ ] 노이즈 카드가 "명백히 틀린 것"이 아니라 "그럴듯하지만 약한 것"인가?
  → O/X로 판별 가능하면 퀴즈가 됨. 판단의 여지가 있어야 게임
- [ ] 시그널을 노이즈로 버렸을 때, "아 그것도 있었구나"가 되는가?
  → 단순 오답 처리가 아니라 관점의 차이로 느껴져야 함

### 톤 테스트
- [ ] 교훈 착지가 없는가? (결론 문장에 "~해야 합니다" 류)
- [ ] 설명조가 없는가? ("이것은 ~를 의미합니다" 류)
- [ ] 또래 말투인가? (교과서/보고서 톤이 아닌가)
- [ ] AI 냄새가 나지 않는가? (자연스러운 문장인가)

## 3. 교육적 검증 (심층 — 주요 라운드)

### 사고 전환 테스트
"이 게임을 하기 전과 후에, 학생이 AI 뉴스를 만났을 때 하는 행동이 달라지는가?"

- [ ] 전: 헤드라인만 보고 반응 → 후: "이건 어떤 종류의 신호지?" (분류 습관)
- [ ] 전: 그럴듯하면 믿음 → 후: "출처가 뭐지?" (검증 습관)
- [ ] 전: "AI는 좋다/나쁘다" → 후: "나는 이 이슈를 이렇게 본다" (자기 프레임)

### 모델1과의 보완 테스트
- [ ] 모델1이 쓰는 근육(감정/공감/자기인식)과 **겹치지 않는** 근육을 쓰는가?
- [ ] 모델1과 모델2를 같은 학생이 둘 다 했을 때, "다른 경험"이라고 느끼는가?

### 에이전시 테스트
- [ ] 학생이 "내가 만든 것"이라는 소유감을 느끼는가? (브리핑 = 내 것)
- [ ] 정답이 없다는 것이 불안이 아니라 자유로 느껴지는가?
- [ ] "다시 해보고 싶다"는 동기가 생기는가? (다른 신호를 골라보고 싶다)

## 4. 디자인 검증 (비주얼 — 주요 라운드)

### 놀공 디자인 원칙
- [ ] 흰 배경 기본, 탁한 색 없음
- [ ] 흑백 + 신호 컬러(🔵🟡🔴🟣)만 포인트
- [ ] 80년대 매킨토시 픽셀 느낌 (모델1과 일관성)
- [ ] 미니멀 — 장식 없음, 정보만
- [ ] 생동감은 컬러가 아니라 애니메이션으로

---

## 5. 최종 통과 기준

아래 3개를 **모두** 만족해야 최종 후보로 제출:

1. **거울이 작동한다** — 플레이어마다 다른 브리핑, 다른 유형
2. **게임이다** — 숙제나 퀴즈가 아니라, "한 번 더 해보고 싶다"
3. **15분에 끝난다** — 교실에서 쓸 수 있는 분량

## 루프 운영 규칙

### 라운드 기록
각 라운드의 결과를 `runs/run_N.md`에 기록:
- 이번 라운드에서 시도한 것
- 위 체크리스트 결과
- 병목 (가장 문제인 부분)
- 다음 라운드에서 바꿀 것

### 라운드 수
- 기획 루프: 3~5라운드
- 프로토타입 루프: 3~5라운드
- 총 최대 10라운드 이내에 최종 후보 2개 제출

### 피터공 개입 시점
- Canon 파일 승인 후 → 자율 루프 시작
- 루프 중간 개입 없음 (Constitution 기준으로 자체 판단)
- 최종 후보 2개 제출 시 → 피터공 검토
