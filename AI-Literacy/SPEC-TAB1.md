## 핵심 컨셉

**4컷 만화 — 좌에서 우로 한 컷씩 채워진다.**

모바일 세로 스크롤(v03) 대신, 넓은 화면에서 패널이 가로로 추가되며 이야기가 펼쳐지는 구조.
선택은 2번. 1컷과 2컷에서 고르면 3~4컷은 결과가 흘러간다.

## 화면 구조

### 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  [Act 표시]          [스탯 바: 중심잡기|융합하기|성찰하기] │
├────────────┬────────────┬────────────┬────────────┤
│            │            │            │            │
│   1컷      │   2컷      │   3컷      │   4컷      │
│  (상황)    │ (전개)     │ (대면)     │ (엔딩)     │
│            │            │            │            │
├────────────┴────────────┴────────────┴────────────┤
│  [선택지 영역 — 현재 컷 아래 또는 옆에 표시]          │
└─────────────────────────────────────────────────────────┘
```

### 패널 사이즈

- 컨테이너: `max-width: 1200px`, 중앙 정렬
- 패널 4개: 각 `25%` 폭, 고정 높이 (aspect-ratio 3:4 또는 유사)
- 패널 간 간격: 12~16px
- 패널 스타일: 둥근 모서리(8px), 얇은 보더(1px #ddd), 약간의 그림자

### 패널 내부 구조

```
┌──────────────┐
│  [이미지]    │ ← 상단 40~50%, 이미지 슬롯 (placeholder → 나중에 교체)
│              │
├──────────────┤
│  텍스트      │ ← 하단 50~60%, 시나리오 텍스트
│              │
│  (선택표시)  │ ← 완료된 패널: 선택한 것 작게 표시
└──────────────┘
```

- 이미지 슬롯: 패널 상단, 회색 placeholder (border: dashed). `<img>` 태그로 나중에 교체 가능
- 이미지 없을 때: placeholder에 컷 번호 + 간단한 아이콘/라벨

### 패널 상태

| 상태 | 시각 표현 |
|------|----------|
| 비어있음 | 점선 보더, 연한 회색 배경, 컷 번호만 표시 |
| 활성 (현재 컷) | 실선 보더, 흰 배경, 텍스트 표시 |
| 완료 | 실선 보더, 흰 배경, 텍스트 + 선택 결과 표시 |

## 인터랙션 흐름

### Act 시작

1. Act 전환 카드: 화면 중앙에 "Act 1 — 숙제 + ChatGPT" + 부제 표시
2. 클릭/탭 → 4컷 프레임 등장 (1컷만 채워짐, 2~4컷은 빈 패널)

### 1컷 — 상황 제시

1. 1컷 패널에 상황 텍스트 표시 (타이핑 효과 또는 페이드인)
2. 1컷 아래 또는 오른쪽에 **선택지 3개** 카드로 표시
3. 각 선택지: 카드 형태, 호버 시 약간 올라옴
4. 선택지에 스탯 프리뷰 **안 보여줌** (v03 디렉션 유지)

### 선택 → 2컷

1. 선택한 카드 강조 (0.3초) → 선택지 영역 사라짐
2. 1컷 패널에 선택 결과 간단히 표시 (선택한 텍스트가 패널 하단에 작게)
3. **2컷 패널 슬라이드인** — 왼쪽에서 밀려들어오거나 페이드인
4. 2컷 텍스트 표시 + 새 선택지 등장
5. 스탯 바 변동 애니메이션 (reasons 텍스트 포함)

### 선택 → 3컷

1. 같은 패턴. 2컷 확정 → 3컷 패널 등장
2. 3컷은 대부분 "다음" 자동 진행 (선택 없음)
3. 일부 루트(hw_present_blind)만 2지선다

### → 4컷 (엔딩)

1. 3컷 확정 → 4컷 패널 등장
2. 엔딩 텍스트 표시
3. 결과 톤에 따라 패널 배경색 미세 변화:
   - good: 연한 초록 틴트
   - neutral: 그대로 (흰)
   - bad: 연한 빨강 틴트
4. **핵심 문장** (highlight) 볼드 표시

### 4컷 완성 후

1. 잠시 전체 4컷을 조망 (0.5초)
2. 스탯 변동 요약 표시 (이 Act에서 얼마나 변했는지)
3. "다음 상황으로 →" 버튼

### Act 간 전환 — 바인더 노트북 구분선 (4/6 결정)

**완료된 4컷을 지우지 않는다.** 아래로 계속 추가.

Act 사이에 **매킨토시 SE 스타일 바인더 구분선** 삽입:
- 가로 구분 영역 (높이 ~80px)
- 좌측에 바인더 링 홀 3개 (1bit 픽셀아트 원형, 세로 배열)
- 구분선 중앙에 Act 타이틀 ("Act 2 — 독후감")
- 배경: 약간의 노트 텍스처 느낌 (연한 수평선 패턴 또는 점선)
- 전체 톤: Classic Mac SE — 1bit 흑백, 픽셀 느낌, Chicago 폰트 스타일

```
──── ○ ──────── Act 2 — 독후감 ──────────────
     ○   "어린 왕자를 읽었다. 솔직히 잘 모르겠다."
     ○
─────────────────────────────────────────────
```

### 스크롤 구조 변경

- 전체 페이지가 **세로 스크롤** (Act 1 → 구분선 → Act 2 → 구분선 → Act 3)
- 완료된 Act는 위에 남아있고 새 Act가 아래에 추가됨
- 새 Act 추가 시 자동 스크롤다운
- 스탯 바: 상단 **sticky** (스크롤해도 항상 보임)

### 전체 완료 후

- 3개 Act 모두 완료 → 최종 프로필 화면 (페이지 맨 아래)
- 누적 스탯 바 차트 + 성찰 코멘트 (v03과 동일)

## 선택지 UI

### 표시 방식

선택지는 현재 활성 패널 **오른쪽**에 세로로 나열:

```
┌──────────┬────────┬──────────┬──────────┬──────────┐
│  1컷     │선택 A  │  (빈)    │  (빈)    │  (빈)    │
│  상황    │선택 B  │          │          │          │
│  텍스트  │선택 C  │          │          │          │
└──────────┴────────┴──────────┴──────────┴──────────┘
```

- 선택지 카드: 활성 패널 오른쪽에 세로 스택
- **선택지 카드 폭 = 패널 폭과 동일** (4/6 결정)
- 선택지 영역 폭: 패널 1개 폭 정도 (빈 패널 자리를 활용)
- 호버: 카드 살짝 오른쪽으로 + 그림자 강화
- 클릭: 카드 강조 → 나머지 페이드아웃 → 선택지 영역이 다음 패널로 전환

### 2지선다일 때

카드 2개만 표시. 세로 중앙 정렬.
